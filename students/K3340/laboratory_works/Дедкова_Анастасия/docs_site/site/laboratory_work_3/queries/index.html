    Кастомные запросы

    Ниже приведены API-запросы, реализованные специально для библиотекаря.

    1. Какие книги закреплены за заданным читателем?

    GET /api/readers/<id>/loans/

    2. Кто из читателей взял книгу более месяца тому назад?

    GET /api/readers/overdue/

    3. Читатели младше 20 лет

    GET /api/readers/younger-than-20/

    4. За кем из читателей закреплены книги, количество экземпляров которых в библиотеке не превышает 2?

    GET /api/readers/rare-books/

    5. Статистика по образованию

    GET /api/readers/education-stats/

    Ответ:

    {
  "primary": 12,
  "secondary": 40,
  "higher": 28,
  "academic_degree": 3
}

    6. Отчёт о работе библиотеки за месяц

    URL:
    GET /api/reports/monthly/

    Параметры запроса (query params):

    • year — год отчётного периода (по умолчанию — текущий год);
    • month — номер месяца (1–12, по умолчанию — текущий месяц).

    Если параметры не указаны, используется текущий месяц.

    Пример запроса

    GET /api/reports/monthly/?year=2025&month=11

    Пример ответа (сокращённо):

    {
  "year": 2025,
  "month": 11,
  "per_day": [
    {
      "date": "2025-11-01",
      "halls": [
        {
          "hall_id": 1,
          "hall_name": "Главный зал",
          "books": 15,
          "readers": 20
        }
      ],
      "total_books": 15,
      "total_readers": 20
    }
  ],
  "new_readers": {
    "total": 6,
    "by_hall": [
      {
        "hall_id": 1,
        "hall_name": "Главный зал",
        "count": 3
      }
    ]
  }
}

    Значение полей:

    • per_day — список дней месяца;

    • halls — по каждому залу: количество экземпляров книг и читателей на этот день;

    • total_books, total_readers — суммарные значения по библиотеке в целом.

    • new_readers — статистика регистрации за месяц:

    • total — сколько человек записалось в библиотеку;

    • by_hall — сколько читателей зарегистрировалось в каждый зал.

    Swagger UI (Интерактивная документация API)

    Swagger подключён через библиотеку drf-yasg.

    http://127.0.0.1:8000/swagger/

    Он предоставляет удобный визуальный интерфейс для тестирования всего API проекта: создания, обновления, удаления объектов и проверки кастомных аналитических запросов.

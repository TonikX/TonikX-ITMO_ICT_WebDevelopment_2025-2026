# Структура клиентского приложения

Проект выполнен на Vue 3 и имеет следующую структуру:

```
src/
├─ api/
│ └─ client.js – настроенный Axios
├─ views/ – страницы приложения
├─ components/ – общие компоненты (при необходимости)
├─ stores/
│ └─ auth.js – Pinia-хранилище пользователя
├─ router/
│ └─ index.js – маршруты приложения
├─ App.vue – каркас приложения (шапка + RouterView)
└─ main.js – инициализация Vue и Pinia
```

---

## Хранилище (Pinia)

Используется одно хранилище `auth`:
- хранит токен доступа;
- хранит данные пользователя;
- определяет роль (админ или нет);
- автоматически подставляет токен в Axios.

При обновлении страницы пользователь остаётся авторизованным.

---

## Маршрутизация

Каждая страница — это Vue-компонент в `views/`.

Маршруты разделены:
- **общедоступные**: каталог книг;
- **только авторизованные**: профиль;
- **только администратор**: выдача, экземпляры, отчёты.

В `router/index.js` добавлены защиты маршрутов:
- если нет токена → перенаправление на `/login`;
- если роль не admin → скрытие админских страниц.

---

## Запросы к API

Взаимодействие происходит через `api/client.js`.

Поддерживаются запросы:
- авторизация и регистрация через Djoser;
- загрузка списка книг, залов, экземпляров;
- выдача и возврат книг;
- поиск читателя по билету;
- отчёты.

Каждая страница сама вызывает свои запросы.

---

## Взаимодействие страниц

- `App.vue` содержит шапку и меню.  
  Пункты меню динамически отображаются в зависимости от роли пользователя.

- Внутри `main-container` отрисовывается текущая страница (`<RouterView />`).

Таким образом, приложение работает как полноценный SPA без перезагрузки.

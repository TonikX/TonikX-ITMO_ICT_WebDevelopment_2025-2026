# Интерфейс и функционал приложения

Ниже перечислены все основные страницы и что на них реализовано.

---

## Каталог книг (`/books`)
- отображение полного списка книг;
- данные загружаются через `/books/`;
- доступно всем пользователям, вход необязателен.

---

## Профиль читателя (`/profile`)
Пользователь вводит:
- номер читательского билета;
- номер паспорта.

Страница показывает:
- ФИО;
- зал;
- образование;
- активные выдачи с подсветкой просрочек.

Пользователь может изменить:
- телефон,
- адрес.

Данные обновляются через PATCH-запрос.

---

## Книги на руках (`/on-loan`) - только администратор
Есть таблица всех активных выдач:
- ФИО читателя;
- название книги;
- сколько дней на руках;
- подсветка просроченных.

---

## Выдача книг (`/manage-loans`) - только администратор
Функции:
- выбрать читателя;
- выбрать книгу;
- выбрать экземпляр;
- выдать книгу;
- вернуть выданную книгу.

Все действия отправляются через POST/PATCH.

---

## Управление экземплярами (`/manage-copies`) - только администратор
Админ может:
- добавить новый экземпляр книги (книга + зал + дата поступления);
- просматривать таблицу всех экземпляров;
- фильтровать по статусу;
- списывать экземпляры с указанием даты.

---

## Отчёты (`/reports`) - только администратор
Доступны два отчёта:
1. **Распределение читателей по образованию**
2. **Новые читатели по залам за месяц**

Данные приходят с бэкенда и отображаются в виде таблиц.

---

## Навигация
Шапка приложения автоматически изменяется:
- если пользователь не авторизован - показываются только "Книги", "Вход", "Регистрация";
- если авторизован - появляется "Профиль";
- если админ - появляются "Выдача книг", "Экземпляры", "Отчёты", "Админка".

---

## Итог
В результате реализовано полноценное клиентское приложение, закрывающее все функции библиотеки от просмотра книг до работы администратора.

# Интерфейсы

## Авторизация

### Login (`/login`)

Форма входа: username + password. При успехе токен сохраняется в localStorage, редирект на главную.

### Register (`/register`)

Форма регистрации: username, email, password. После регистрации перенаправляет на логин.

### Profile (`/profile`)

Просмотр и редактирование данных текущего пользователя (email, имя, фамилия).

---

## CRUD-интерфейсы

Все CRUD-страницы используют общий composable `useCrud` и имеют единую структуру:

- Таблица (`v-data-table`) со списком записей
- Кнопка добавления
- Диалог с формой для создания/редактирования
- Кнопки редактирования и удаления в каждой строке

### Drivers (`/`)

Управление водителями. Поля: имя, фамилия, паспорт, класс, стаж, оклад, автобус, маршрут.

### Buses (`/buses`)

Управление автобусами. Поля: гос. номер, тип автобуса.

### Bus Types (`/bus-types`)

Управление типами автобусов. Поля: название, вместимость.

### Routes (`/routes`)

Управление маршрутами. Поля: номер, начальный/конечный пункт, время начала/конца, интервал, длительность.

### Schedules (`/schedules`)

График работы водителей. Поля: водитель, автобус, маршрут, дата, начало/конец смены.

### Absences (`/absences`)

Автобусы, не вышедшие на линию. Поля: автобус, дата, причина, примечание.

---

## Отчет (`/report`)

Страница с аналитикой:

- **Общая протяженность маршрутов** — суммарная длительность в минутах
- **Водители по классам** — количество водителей каждого класса
- **Время движения** — начало и конец движения по каждому маршруту
- **Отчет по типам автобусов** — раскрывающиеся панели с маршрутами, автобусами и водителями для каждого типа

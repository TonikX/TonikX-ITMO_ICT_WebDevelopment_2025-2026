## Управление состоянием

Для централизованного управления состоянием авторизации в клиентском приложении используется библиотека Pinia, которая
является официальным инструментом управления состоянием во Vue 3.

Хранилище авторизации реализовано в файле store/auth.js.

### Назначение хранилища

Данное хранилище предназначено для:

* хранения токена авторизации;
* хранения данных текущего пользователя;
* определения статуса аутентификации;
* получения информации о пользователе с серверной части;
* выполнения выхода из системы.
* Использование Pinia позволяет получить доступ к этим данным из любого компонента приложения.

Структура состояния (state)

```js
state: () => ({
    token: localStorage.getItem("token"),
    user: null,
    isLoading: false,
}),
```

### Описание полей состояния:

* token — токен авторизации, сохраняемый в localStorage для сохранения сессии между перезагрузками страницы;
* user — объект с данными текущего пользователя;
* isLoading — флаг, указывающий на процесс загрузки данных пользователя.

### Геттеры (getters)

```js
getters: {
    isAuthenticated: (state) => !!state.token,
}
,
```

Геттер isAuthenticated используется для определения, авторизован ли пользователь в системе. Он возвращает логическое
значение на основе наличия токена.

Данный геттер применяется, в частности, при защите маршрутов в Vue Router.

### Действия (actions)

Установка токена

```js
setToken(token)
{
    this.token = token;
    localStorage.setItem("token", token);
}
```

Метод сохраняет токен в состоянии приложения и в localStorage, обеспечивая сохранение авторизации после обновления
страницы.

Установка данных пользователя

```js
setUser(user)
{
    this.user = user;
}
```

Используется для сохранения информации о текущем пользователе после успешной авторизации или обновления данных профиля.

### Обновление данных пользователя

```js
updateUser(updatedUserData)
{
    if (this.user) {
        this.user = {...this.user, ...updatedUserData};
    }
}
```

Позволяет обновлять отдельные поля данных пользователя без повторного запроса к серверу.

### Получение текущего пользователя

```js
async
fetchUser()
{
    if (!this.token) return;

    this.isLoading = true;
    try {
        const user = await getCurrentUser(this.token);
        this.user = user;
    } catch (err) {
        this.logout();
    } finally {
        this.isLoading = false;
    }
}
```

Метод выполняет запрос к серверу для получения информации о текущем пользователе на основе сохранённого токена.

В случае ошибки (например, истёкший токен) выполняется выход из системы.

### Выход из системы

```js
logout()
{
    this.token = null;
    this.user = null;
    localStorage.removeItem("token");
}
```

Метод очищает данные пользователя и удаляет токен из localStorage, полностью завершая пользовательскую сессию.

### Интеграция с другими частями приложения

Хранилище auth используется в следующих частях приложения:

* при защите маршрутов (Vue Router);
* при выполнении запросов к серверу;
* при отображении информации о текущем пользователе;
* при выходе из системы.

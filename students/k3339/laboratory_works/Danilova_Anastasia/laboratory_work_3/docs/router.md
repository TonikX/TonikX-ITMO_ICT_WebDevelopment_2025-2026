## Маршрутизация

Для организации навигации в клиентской части приложения используется библиотека Vue Router, предназначенная для
реализации маршрутизации в одностраничных приложениях (SPA), разработанных с использованием Vue.js.

Описание маршрутов

В приложении реализованы следующие маршруты:

| Путь                | Название маршрута | Компонент        | Описание                                                    |
|---------------------|-------------------|------------------|-------------------------------------------------------------|
| `/`                 | home              | HomeView         | Главная страница приложения с краткой информацией о системе |
| `/login`            | login             | LoginView        | Страница авторизации пользователей в системе                |
| `/register`         | register          | RegisterView     | Страница регистрации новых пользователей                    |
| `/dashboard`        | dashboard         | DashboardView    | Основная панель управления пользователя со списком объектов |
| `/settings`         | UserSettings      | UserSettingsView | Страница настроек пользователя и редактирования профиля     |
| `/objects/:id`      | object-detail     | ObjectDetailView | Просмотр детальной информации об объекте (парке)            |
| `/objects/:id/edit` | object-edit       | ObjectEditView   | Редактирование информации об объекте                        |
| `/objects/new`      | object-create     | ObjectCreateView | Создание нового объекта (парка)                             |

### Защищенные маршруты

Для ограничения доступа к определённым страницам используется мета-поле requiresAuth:

```js
meta: {
    requiresAuth: true
}
```

### Глобальный навигационный guard

Для проверки прав доступа реализован глобальный навигационный guard beforeEach:

```js
router.beforeEach((to, from, next) => {
    const authStore = useAuthStore();

    if (to.meta.requiresAuth && !authStore.isAuthenticated) {
        next("/login");
    } else {
        next();
    }
});
```

### Логика работы guard’а:

* если маршрут требует авторизации, а пользователь не авторизован — выполняется перенаправление на страницу входа (
  /login);
* если пользователь имеет доступ — переход разрешается;
* проверка осуществляется на основе состояния авторизации, хранящегося в Pinia (authStore.isAuthenticated).

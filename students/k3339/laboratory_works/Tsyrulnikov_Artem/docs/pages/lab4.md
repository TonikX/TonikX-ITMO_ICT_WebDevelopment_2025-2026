## Лабораторная работа: Vue.js клиент для рекламного агентства

В четвёртой лабораторной работе реализован клиентский интерфейс на Vue.js 3 с Vuetify для взаимодействия с REST API из лабораторной работы №3. Проект включает авторизацию по JWT, CRUD-операции для всех сущностей и отчёт за квартал.

## Вариант задания

- Настроить CORS для серверной части из лабораторной работы №3.
- Реализовать интерфейсы авторизации, регистрации и изменения учётных данных.
- Создать клиентские интерфейсы для работы с данными рекламного агентства.
- Подключить Vuetify для стилизации компонентов.

## Ход выполнения

## Задание 1: Настройка CORS в Django

_Установка_

- Добавлен пакет `django-cors-headers` в зависимости.
- В `config/settings.py`: добавлен `corsheaders` в `INSTALLED_APPS`, middleware `CorsMiddleware` после `SessionMiddleware`.

_Конфигурация_

- `CORS_ALLOWED_ORIGINS`: разрешены запросы с `localhost:5173` (Vite dev server).
- `CORS_ALLOW_CREDENTIALS = True` для передачи cookies.

## Задание 2: Структура Vue.js проекта

_Технологии_

- Vue.js 3 с Composition API (`<script setup>`).
- Vite для сборки и dev-сервера.
- Pinia для управления состоянием.
- Vue Router для маршрутизации.
- Vuetify 3 для UI-компонентов.
- Axios для HTTP-запросов.

_Структура проекта_

```
Lab4/
├── src/
│   ├── main.js           # точка входа
│   ├── App.vue           # корневой компонент с навигацией
│   ├── api/index.js      # axios с JWT interceptors
│   ├── plugins/vuetify.js
│   ├── router/index.js   # маршруты с guards
│   ├── stores/auth.js    # Pinia store авторизации
│   └── views/            # страницы приложения
├── package.json
└── vite.config.js
```

## Задание 3: Авторизация и регистрация

_Pinia Store_

- `stores/auth.js`: хранит состояние `user` и `isAuth`, методы `login`, `register`, `logout`, `fetchUser`, `updateUser`.
- Токены сохраняются в `localStorage` (access и refresh).

_JWT Interceptors_

- `api/index.js`: автоматическое добавление `Authorization: Bearer` к запросам.
- При 401 ошибке — попытка refresh токена, при неудаче — редирект на `/login`.

_Страницы_

- **Login.vue** — форма входа с валидацией и отображением ошибок.
- **Register.vue** — форма регистрации, после успеха — автоматический вход.
- **Profile.vue** — редактирование имени, фамилии и email текущего пользователя.

_Router Guards_

- Защита маршрутов через `meta.auth` — неавторизованные перенаправляются на `/login`.

## Задание 4: Интерфейсы для работы с данными

_Общий паттерн_

- Каждая страница использует `v-data-table` для отображения списка.
- Диалоговое окно `v-dialog` для создания/редактирования записей.
- Кнопки редактирования и удаления в каждой строке таблицы.

_Реализованные страницы_

- **Clients.vue** — CRUD для клиентов (название, контактное лицо, телефон, email).
- **Services.vue** — CRUD для услуг с выбором категории через `v-select`.
- **Orders.vue** — CRUD для заказов с выбором клиента, услуги, исполнителя и статуса.
- **Payments.vue** — CRUD для платежей с чекбоксом статуса оплаты и `v-chip` для визуализации.

_Особенности_

- Связанные сущности (клиент, услуга, исполнитель) загружаются параллельно через `Promise.all`.
- Отображение имён вместо ID через поиск в загруженных массивах.

## Задание 5: Главная страница и отчёты

_Статистика_

- **Home.vue** показывает 4 карточки с количеством: клиенты, услуги, заказы, платежи.
- Данные загружаются параллельно при монтировании компонента.

_Квартальный отчёт_

- Таблица с исполнителями и суммарной стоимостью работ за последние 3 месяца.
- Использует эндпоинт `/api/payments/quarter_report/`.
- Внизу таблицы — итоговая сумма через `computed`.

_Пагинация_

- В Django добавлена глобальная пагинация (10 элементов на страницу).
- Фронтенд обрабатывает ответы формата `{count, results}`.

## Задание 6: Навигация и UI

_App Bar_

- Навигационная панель с кнопками: Главная, Клиенты, Услуги, Заказы, Платежи, Профиль, Выход.
- Отображается только для авторизованных пользователей.

_Vuetify компоненты_

- `v-app`, `v-app-bar`, `v-main`, `v-container` — layout.
- `v-card`, `v-data-table`, `v-dialog` — контент.
- `v-text-field`, `v-select`, `v-checkbox`, `v-btn` — формы.
- `v-chip`, `v-alert` — визуализация статусов и сообщений.

## Выводы

В работе освоил разработку SPA на Vue.js 3: Composition API, управление состоянием через Pinia, маршрутизацию с guards, интеграцию с REST API через axios с JWT-авторизацией. Реализован полный CRUD для всех сущностей рекламного агентства с использованием компонентов Vuetify и поддержкой пагинации.

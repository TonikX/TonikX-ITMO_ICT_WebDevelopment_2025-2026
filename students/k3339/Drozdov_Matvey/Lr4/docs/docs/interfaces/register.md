# Страница регистрации (Register)

## Структура интерфейса

Страница представляет собой форму регистрации с вертикальным расположением элементов.

### Поля формы

#### Email

- тип поля: email
- используется для контактных данных пользователя
- поддерживает стандартный email autocomplete

---

#### Логин (username)

- текстовое поле
- обязательное
- должно быть уникальным

---

#### Пароль (password)

- поле типа password
- скрывает вводимые символы
- обязательное

---

## Кнопки управления

### Создать

Назначение:

- отправляет данные регистрации на сервер

Состояния:

- Создать — стандартное состояние
- Создаю... — во время выполнения запроса

Кнопка блокируется во время отправки запроса.

---

## Навигация

### Ссылка "Уже есть аккаунт? Войти"

Перенаправляет пользователя на страницу входа.

---

## Поведение интерфейса

### Успешная регистрация

1. Пользователь вводит данные.
2. Отправляется запрос создания пользователя.
3. Отображается сообщение:

   "Аккаунт создан. Теперь можно войти."

4. Через короткую задержку выполняется переход на страницу входа.

---

### Ошибки регистрации

Если сервер возвращает ошибки (например от Djoser), интерфейс:

- отображает ошибки под формой
- показывает сообщения по каждому полю

Пример:

username: пользователь уже существует
password: пароль слишком простой

---

## Логика состояния

Используются reactive состояния:

- loading — блокирует форму во время запроса
- success — отображает успешное создание аккаунта
- error — содержит текст ошибки

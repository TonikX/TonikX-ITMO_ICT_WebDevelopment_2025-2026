# Описание моделей данных

## Основные сущности

### User

Пользователь платформы. Может быть как обычным пользователем, так и автором контента.

**Поля:**

- `id` — уникальный идентификатор (первичный ключ)
- `name` — имя пользователя (до 128 символов)
- `email` — электронная почта (уникальная, до 64 символов)
- `password_hash` — хешированный пароль (до 512 символов)
- `created_at` — время создания аккаунта
- `updated_at` — время последнего обновления профиля

**Связи:**

- `likes` — лайки, поставленные пользователем
- `comments` — комментарии пользователя
- `subscriptions` — подписки пользователя на авторов
- `author_profile` — профиль автора (если пользователь является автором)

---

### Author

Профиль автора контента. Связан с пользователем отношением один-к-одному.

**Поля:**

- `id` — уникальный идентификатор
- `user_id` — ссылка на пользователя (уникальная)
- `name` — имя автора (до 256 символов, необязательно)
- `handle` — юзернейм/ручка автора (до 256 символов, обязательно)
- `bio` — биография автора (до 256 символов, необязательно)
- `is_verified` — статус верификации
- `created_at` — дата создания профиля
- `updated_at` — дата последнего обновления

**Связи:**

- `contents` — медиаконтент автора (фото/видео)
- `posts` — посты автора
- `subscribers` — подписчики автора
- `author_profile` — ссылка на пользователя

---

### Post

Пост, опубликованный автором. Может быть бесплатным или платным.

**Поля:**

- `id` — уникальный идентификатор
- `text` — текст поста (до 1024 символов, необязательно)
- `author_id` — ссылка на автора
- `is_free_post` — флаг доступности (true = бесплатный, false = платный)
- `likes_count` — вычисляемое поле: количество лайков
- `comments_count` — вычисляемое поле: количество комментариев
- `created_at` — дата публикации
- `updated_at` — дата последнего обновления

**Связи:**

- `author` — автор поста
- `contents` — мультимедиа-контент поста (фото/видео)
- `likes` — лайки на пост
- `comments` — комментарии к посту

---

### Content (полиморфная модель)

Медиаконтент (фото или видео) к посту или профилю автора.

**Базовые поля:**

- `id` — уникальный идентификатор
- `type` — тип контента (photo или video)
- `author_id` — ссылка на автора (необязательно)
- `post_id` — ссылка на пост (необязательно)
- `created_at` — дата загрузки
- `updated_at` — дата последнего обновления

**Подклассы:**

#### Photo

Изображение.

- `width` — ширина (пиксели, необязательно)
- `height` — высота (пиксели, необязательно)

#### Video

Видеоролик.

- `duration` — длительность (секунды, необязательно)

**Связи:**

- `author` — автор контента
- `post` — пост, к которому относится контент

---

### Like

Лайк пользователя на пост. Уникален по паре (user_id, post_id).

**Поля:**

- `id` — уникальный идентификатор
- `user_id` — ссылка на пользователя
- `post_id` — ссылка на пост
- `created_at` — время лайка
- `updated_at` — время последнего обновления

**Связи:**

- `user` — пользователь, поставивший лайк
- `post` — пост, получивший лайк

---

### Comment

Комментарий пользователя к посту.

**Поля:**

- `id` — уникальный идентификатор
- `text` — текст комментария (до 1024 символов)
- `user_id` — ссылка на пользователя
- `post_id` — ссылка на пост
- `created_at` — дата написания
- `updated_at` — дата последнего редактирования

**Связи:**

- `user` — автор комментария
- `post` — пост, к которому оставлен комментарий

---

### Subscription

Подписка пользователя на автора. Может быть бесплатной или платной.

**Поля:**

- `id` — уникальный идентификатор
- `subscriber_id` — ссылка на подписчика (пользователя)
- `author_id` — ссылка на автора
- `started_at` — дата начала подписки (по умолчанию текущее время)
- `expires_at` — дата окончания подписки (null = бесконечная)
- `renewable` — флаг автоматического продления (по умолчанию true)
- `created_at` — дата оформления подписки
- `updated_at` — дата последнего обновления

**Связи:**

- `subscriber` — пользователь, оформивший подписку
- `author` — автор, на которого подписан пользователь

**Ограничение:** уникальная подписка на пару (subscriber_id, author_id)

---

## Диаграмма связей

```
User (1) ──── (∞) Like
User (1) ──── (∞) Comment
User (1) ──── (∞) Subscription
User (1) ──── (0..1) Author

Author (1) ──── (∞) Post
Author (1) ──── (∞) Content
Author (1) ──── (∞) Subscription

Post (1) ──── (∞) Like
Post (1) ──── (∞) Comment
Post (1) ──── (∞) Content

Content — полиморфная: Photo | Video
```

# LAB3WEB — Отчет по проекту

## 1. Общая информация

**Проект:** LAB3WEB (REST API)

**Тема:** Система управления птицефабрикой с аналитическими запросами

**Технологический стек:**
- Python 3.11
- FastAPI
- SQLAlchemy (ORM)
- PostgreSQL
- Pydantic v2
- JWT Bearer Authentication

Проект реализует REST API для управления сущностями птицефабрики (породы, куры, клетки, цеха, рационы и сотрудники), а также набор аналитических запросов, построенных на агрегациях данных.

---

## 2. Архитектура проекта

Проект построен по классической слоистой архитектуре:

```
src/
├── api/        # HTTP-ручки (FastAPI routers)
├── service/    # Бизнес-логика и аналитические запросы
├── models/     # SQLAlchemy ORM модели
├── schemas/    # Pydantic-схемы (request/response)
├── database.py # Инициализация SessionLocal
├── deps/       # Зависимости (auth, db)
└── main.py     # Точка входа FastAPI
```

**Принцип:**
- `api` — не содержит логики, только валидацию и вызов сервисов
- `service` — вся работа с ORM и агрегациями
- `schemas` — контракт API

---

## 3. Модель данных

### Основные сущности

#### Breed (Порода)
- `breed_id` (PK)
- `name`
- `avg_eggs_per_month`
- `avg_weight_kg`
- `recommended_diet_no`

Связи:
- 1 → N с `Chicken`
- 1 → N с `BreedDietSeason`

---

#### Chicken (Курица)
- `chicken_id` (PK)
- `breed_id` (FK)
- `cage_id` (FK)
- `weight_kg`
- `age_months`
- `eggs_per_month`

Связи:
- N → 1 с `Breed`
- N → 1 с `Cage`
- 1 → N с `ChickenMove`

---

#### Cage (Клетка)
- `cage_id` (PK)
- `workshop_id` (FK)
- `row_no`
- `cage_no`

Связи:
- N → 1 с `Workshop`
- 1 → N с `Chicken`

---

#### Workshop (Цех)
- `workshop_id` (PK)
- `name`

Связи:
- 1 → N с `Cage`

---

#### Diet / BreedDietSeason
Используются для хранения сезонных рационов для пород.

## 4. Аутентификация и безопасность

- Используется **Bearer Token Authentication (JWT)**
- Все аналитические и CRUD-ручки защищены зависимостью:

```python
Depends(get_current_user)
```

- Неавторизованный доступ к API невозможен

---

## 5. CRUD-функциональность

Реализованы стандартные CRUD-операции для:
- Пород (`Breed`)
- Кур (`Chicken`)
- Клеток (`Cage`)
- Цехов (`Workshop`)
- Рационов (`Diet`)

CRUD используется **только для управления данными**, а не для аналитики.

---

## 6. Аналитические запросы

Аналитика реализована **отдельно от CRUD**, что соответствует требованиям задания.

### Принципы аналитики:
- Только `SELECT`
- Использование агрегаций (`AVG`, `SUM`, вычисляемые поля)
- ORM SQLAlchemy
- Отдельный слой `service/analytics.py`

---

### Пример 1: Аналитика по яйценоскости кур

**Роут:**
```
GET /analytics/eggs-by-chicken
```

**Описание:**
Возвращает список кур с расчетом среднего количества яиц в день.

**Формула:**
```
eggs_per_day = eggs_per_month / 30
```

**Фильтры:**
- `breed_id`
- `min_weight`
- `max_weight`

**Ответ:**
- `chicken_id`
- `breed_id`
- `breed_name`
- `weight_kg`
- `age_months`
- `eggs_per_month`
- `eggs_per_day`

---

### Пример 2: Агрегация по клеткам

**Идея:**
- Подсчет суммарного количества яиц по клетке
- Средний вес кур в клетке

Используется `GROUP BY cage_id`.

---

### Пример 3: Аналитика по породам

**Идея:**
- Фактическая средняя яйценоскость vs эталонная (`Breed.avg_eggs_per_month`)
- Анализ отклонений

---

## 7. Pydantic-схемы

Для аналитики используются отдельные схемы, например:

```python
class ChickenEggsAnalyticsRow(BaseModel):
    chicken_id: int
    breed_id: int
    breed_name: str
    weight_kg: Decimal
    age_months: int
    eggs_per_month: int
    eggs_per_day: Decimal
```

Это гарантирует:
- Четкий контракт API
- Валидацию данных
- Корректный OpenAPI

## 8. Тестирование

Для ручного тестирования используются простые Python-скрипты:

```python
import requests

r = requests.get(
    "http://localhost:8000/analytics/eggs-by-chicken",
    headers={"Authorization": "Bearer <TOKEN>"},
)

print(r.status_code)
print(r.json())
```

Скрипты предназначены для:
- Проверки роботоспособности API
- Быстрой валидации аналитических запросов

---

## 9. Генерация тестовых данных

В проекте присутствует скрипт генерации данных, который:
- Создает породы, цеха, клетки
- Заполняет кур (сотни записей)
- Привязывает сотрудников

Это позволяет полноценно тестировать агрегационные запросы.

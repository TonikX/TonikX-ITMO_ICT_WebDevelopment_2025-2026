# ОТЧЕТ  
по практическим занятиям №3.1 и №3.2  
**Тема: Работа с REST API, ORM и базой данных**

---

## 1. Цель работы

Целью практической работы является изучение принципов REST-архитектуры, освоение работы с ORM для выполнения запросов к базе данных, а также разработка REST API для управления сущностями предметной области с использованием современных Python-фреймворков.

---

## 2. Используемые технологии

В ходе выполнения практической работы были использованы следующие технологии и инструменты:

- Язык программирования: Python  
- Фреймворк для API: FastAPI  
- ORM: SQLAlchemy (async)  
- База данных: PostgreSQL  
- Сервер приложений: Uvicorn  
- Административная панель: Starlette Admin  
- Формат обмена данными: JSON  

---

## 3. Описание предметной области

В рамках практики была реализована система учета автовладельцев и автомобилей, а также REST API для управления сущностями.

### Основные сущности системы:
- **User** — пользователь (автовладелец)
- **DriverLicense** — водительское удостоверение
- **Car** — автомобиль
- **Ownership** — владение автомобилем

Между сущностями реализованы следующие связи:
- Один пользователь может иметь несколько водительских удостоверений
- Один пользователь может владеть несколькими автомобилями
- Один автомобиль может иметь нескольких владельцев в разные периоды времени

---

## 4. Реализация базы данных и моделей

Для описания таблиц базы данных были реализованы ORM-модели:

- User
- DriverLicense
- Car
- Ownership

Каждая модель содержит первичный ключ, необходимые атрибуты и связи (`relationship`).  
Для обеспечения целостности данных используются внешние ключи и уникальные ограничения.

## 5. Заполнение тестовых данных

## 6. Выполнение запросов (Практика №3.1)

### 6.1 Фильтрация данных

Были реализованы следующие запросы:
- Вывод всех автомобилей марки Toyota
- Поиск автомобилей, принадлежащих пользователям с именем Никита
- Получение случайного пользователя и его водительских удостоверений
- Поиск владельцев автомобилей красного цвета
- Поиск владельцев, у которых год начала владения автомобилем равен 2025

---

### 6.2 Агрегация и группировка

Были реализованы агрегатные запросы:
- Определение самой ранней даты выдачи водительского удостоверения
- Определение последней даты начала владения автомобилем
- Подсчет количества автомобилей для каждого пользователя
- Подсчет количества автомобилей по маркам
- Сортировка пользователей по дате выдачи водительского удостоверения

    Структура проекта

    Ниже описана текущая структура веб-приложения scientific_conferences, реализованного на Django. Каждая директория и файл пояснены с точки зрения их назначения в системе.

    scientific_conferences/        # Корень проекта
├─ conferences/                # Приложение "конференции"
│   ├─ migrations/             # Миграции базы данных для моделей conferences
│   ├─ templates/              # HTML-шаблоны, относящиеся к конференциям
│   ├─ __init__.py
│   ├─ admin.py                # Регистрация моделей в Django admin
│   ├─ apps.py                 # Конфигурация приложения
│   ├─ forms.py                # Django-формы (регистрация на конференцию, отзывы)
│   ├─ models.py               # Модели: конференции, регистрации участников, отзывы и т. д.
│   ├─ urls.py                 # Маршруты приложения conferences
│   ├─ views.py                # Вьюхи (логика страниц/обработчиков запросов)
├─ scientific_conferences/     # Глобальная конфигурация проекта Django
│   ├─ __init__.py
│   ├─ asgi.py                 # Точка входа для ASGI-сервера
│   ├─ settings.py             # Основные настройки проекта
│   ├─ urls.py                 # Главный роутер: подключает urls из приложений
│   ├─ wsgi.py                 # Точка входа для WSGI-сервера (продакшн/деплой)
├─ static/                     # Статические файлы (CSS, JS)
│   ├─ css/
│   └─ js/
├─ templates/                  # Глобальные шаблоны для всего проекта
│   ├─ includes/               # Повторно используемые части интерфейса (пагинация)
│   └─ base.html               # Базовый шаблон (layout сайта: шапка, меню, стили Bootstrap)
├─ users/                      # Приложение "пользователи"
│   ├─ templates/              # Шаблоны страниц аутентификации / профиля
│   ├─ __init__.py
│   ├─ apps.py                 # Конфигурация приложения users
│   ├─ forms.py                # Формы регистрации / авторизации пользователя
│   ├─ urls.py                 # Маршруты (регистрация, логин, личный кабинет)
│   └─ views.py                # Логика регистрации, входа, редактирования данных
├─ manage.py                   # CLI-утилита Django для запуска сервера, миграций и т.д.

    Ключевые компоненты проекта

    1. Приложение conferences/

    Отвечает за предметную область "научные конференции". Здесь должны быть реализованы:

    • описание конференции (название, тематика, место и период проведения, условия участия);
    • регистрация участников / авторов на выступление;
    • редактирование и удаление своих регистраций пользователем;
    • отзывы (комментарии с рейтингом 1–10, автором и датой);
    • отображение списка участников по конференциям.

    Файлы внутри:

    • models.py — описывает основные сущности: Conference, Registration, Review, Place, Topic.
    • views.py — контроллеры: список конференций, детальная страница конференции, создание регистрации и т.д.
    • forms.py — Django-формы для создания/изменения регистрации и отзыва.
    • urls.py — маршруты для страниц конференций.
    • admin.py — регистрация моделей для Django admin.
    • Шаблоны для отображения конференций, форм.

    2. Приложение users/

    Отвечает за пользователей системы.

    Функционал в рамках задания:

    • регистрация нового пользователя;
    • авторизация/вход;
    • отображение отзывов и редактирование/удаление своих регистраций.

    Файлы внутри:

    • forms.py — формы регистрации/логина;
    • views.py — вьюхи для регистрации, логина, лк;
    • urls.py — маршруты для регистрации / входа / выхода;
    • templates/ — HTML-шаблоны страниц входа/регистрации.

    3. Директория scientific_conferences/

    • settings.py — настройки INSTALLED_APPS, БД, статика, шаблоны, авторизация, etc.;
    • urls.py — корневой роутер. Обычно тут подключаются path('conferences/', include(...)) и path('users/', include(...));
    • wsgi.py и asgi.py — точки входа для сервера (gunicorn/daphne и т.д. если когда-то деплоишь);
    • __init__.py — просто делает пакет модулем Python.

    4. Шаблоны на уровне проекта (templates/)

    • base.html — общий каркас HTML. Все остальные страницы расширяют его через {% extends "base.html" %}.
    • includes/_pagination.html — пагинация списка конференций.

    5. Директория static/

    Твоя статика:

    • css/ — стили;
    • js/ — скрипты.

    Статика подключается в base.html через {% load static %} и <link rel="stylesheet" href="{% static 'css/...' %}">.

    6. manage.py

    Утилита, через которую запускается и управляется приложение.

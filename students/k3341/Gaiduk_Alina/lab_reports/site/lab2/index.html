    Веб-сервис «Научные конференции» на Django

    Цель работы

    Разработать веб-сервис на Django для управления информацией о научных конференциях, регистрации участников и публикации отзывов, с поддержкой базовых функций CRUD и пользовательских действий.

    Описание предметной области

    Сервис работает с научными конференциями. Для каждой конференции должны храниться:

    • название конференции;
    • список тематик;
    • место проведения;
    • период проведения (даты начала и окончания);
    • описание конференции;
    • описание места проведения;
    • условия участия.

    Функциональные требования

    1. Регистрация пользователей

    • Возможность регистрации нового пользователя (создание аккаунта).
    • Авторизация / вход в систему.
    • В дальнейшем пользователь выступает как автор/участник конференции.

    2. Работа с конференциями

    • Просмотр списка всех конференций.
    • Просмотр детальной страницы одной конференции.
    • Регистрация пользователя на участие/выступление в выбранной конференции.

    • Пользователь может:

    • просматривать свои регистрации на конференции,

    • редактировать данные своей регистрации,
    • удалять свою регистрацию.

    3. Отзывы (комментарии) к конференциям

    • Пользователь может добавить отзыв к конференции.

    • Для каждого отзыва сохраняются:

    • даты конференции (или ссылка на ту конференцию, о которой отзыв),

    • текст комментария,
    • рейтинг (целое число от 1 до 10),
    • информация о комментаторе (кто оставил отзыв).
    • Отзывы должны быть видны на странице конференции.

    4. Администрирование

    • В Django admin у администратора должна быть возможность:

    • управлять конференциями,

    • просматривать регистрации участников,

    • указывать результат выступления участника:

      • «рекомендован к публикации»
      • «не рекомендован к публикации».

    5. Таблица участников

    • В клиентской части (публичная часть сайта) должна отображаться таблица:

    • конференция → список всех участников (зарегистрированных авторов) с их данными.

    • То есть пользователь может открыть страницу и увидеть, кто участвует в какой конференции.

    Дополнительные требования на максимальный балл

    Для поднятия оценки нужно сделать:

    1. Меню навигации

    2. Красивое адаптивное меню.

    3. Пагинация

    4. Разбивка длинных списков на страницы.

    5. Поиск

    6. Поиск по тем же объектам, для которых сделана пагинация (например, искать конференции по названию/тематике).

    Итог

    В результате должна получиться мини-система управления конференциями, в которой:

    • есть пользователи,
    • есть конференции,
    • есть регистрации на конференции,
    • есть отзывы,
    • админ может отмечать результаты выступлений,
    • на фронте видна читаемая сводка участников по конференциям,
    • плюс навбар, пагинация и поиск.

Этот интерфейс работает через Django view на сервере. Формы отправляются без JavaScript — сервер выполняет логику с токеном и сессией.

{% if message %}
{{ message }}
{% endif %}

Вход

{% csrf_token %}

Действия

{% csrf_token %}
{% csrf_token %}

Текущий токен (сохранённый в сессии)

{% if token %}
Ключ токена: {{ token }}
Если нужно использовать токен с curl/Postman: добавьте заголовок Authorization: Token {{ token }}
{% else %}
Токен не найден в сессии.
{% endif %}

Информация о пользователе

{% if user_info %} 
{{ user_info|safe }}
{% else %}
Информация отсутствует. Нажмите «Показать current user» после входа.
{% endif %}
Примечания:
  • Нужен app rest_framework.authtoken в INSTALLED_APPS и выполненные миграции (python manage.py migrate).
  • Этот view использует Django session auth + DRF Token: сайт создаёт/хранит токен на сервере и в сессии браузера — поэтому JS не требуется.
  • Если хочешь использовать JWT — можно изменить логику view, чтобы хранить access/refresh в сессии аналогично.

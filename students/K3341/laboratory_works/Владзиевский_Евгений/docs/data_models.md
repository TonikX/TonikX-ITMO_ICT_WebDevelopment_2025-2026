# Модели данных

Приложение использует SQLAlchemy в качестве ORM. Ниже приведены основные сущности.

## User

Зарегистрированный пользователь системы.

| Поле | Тип | Описание |
| - | - | - |
| `id` | Integer | Первичный ключ |
| `login` | String | Уникальный логин |
| `password` | String | Хэш пароля |
| `name` | String | Отображаемое имя |
| `about` | String | Биография/описание (опционально) |
| `created_time` | DateTime | Время создания |
| `modified_time` | DateTime | Время последнего изменения |

**Связи:**
- Имеет много `Post`
- Имеет много `Like`
- Имеет много `Comment`
- Имеет много `Image`

## Post

Публикация, созданная пользователем.

| Поле | Тип | Описание |
| - | - | - |
| `id` | Integer | Первичный ключ |
| `title` | String | Заголовок |
| `text` | String | Текст |
| `user_id` | Integer | Внешний ключ на пользователя |
| `created_time` | DateTime | Время создания |

**Связи:**
- Принадлежит `User`
- Имеет много `Image`
- Имеет много `Like`
- Имеет много `Comment`
- `like_count` (вычисляемое)
- `comment_count` (вычисляемое)

## Comment

Комментарий к посту.

| Поле | Тип | Описание |
| - | - | - |
| `id` | Integer | Первичный ключ |
| `text` | String | Текст комментария |
| `post_id` | Integer | Внешний ключ на пост |
| `user_id` | Integer | Внешний ключ на пользователя |

## Like

Лайк на пост. Ограничение уникальности: один лайк от пользователя на один пост.

| Поле | Тип | Описание |
| - | - | - |
| `id` | Integer | Первичный ключ |
| `post_id` | Integer | Внешний ключ на пост |
| `user_id` | Integer | Внешний ключ на пользователя |

## Image

Изображение, загруженное в систему.

| Поле | Тип | Описание |
| - | - | - |
| `id` | Integer | Первичный ключ |
| `hash` | String | Хэш файла |
| `width` | Integer | Ширина в пикселях |
| `height` | Integer | Высота в пикселях |
| `created_by` | Integer | Внешний ключ на автора |

# Структура кода

Проект использует модульную структуру, типичную для FastAPI.

## Дерево каталогов (lab/src)

- **api/**: уровень API.
    - **routers/**: эндпоинты, разделённые по доменам.
        - **auth/**: аутентификация.
        - **posts/**: управление постами.
        - **users/**: профили пользователей.
        - **images/**: обработка изображений.
    - `security.py`: работа с JWT и зависимостями безопасности.
    - `sessions.py`: зависимости для подключения к БД.
    
- **db/**: слой базы данных.
    - `schemas.py`: модели SQLAlchemy.
    - `crud.py`: CRUD-операции.

## Важные блоки

### Зависимость сессии базы
`src/api/sessions.py` — выдаёт сессию БД на запрос и корректно закрывает её.

### Безопасность и авторизация
`src/api/security.py` и `src/api/routers/auth`:
- библиотека `fastapi-jwt` для работы с токенами;
- `access_policy` и `refresh_policy` управляют жизненным циклом токенов.

### Алгоритм рекомендаций
`src/db/schemas.py` (функция PL/pgSQL) вычисляет оценку постов по лайкам, комментариям и давности публикации для выдачи рекомендуемой ленты:

```sql
RETURN C * like_count + X * comment_count - F * EXTRACT(EPOCH FROM time_passed);
```

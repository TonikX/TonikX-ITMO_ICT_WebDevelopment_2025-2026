# Миграции, ENV, GitIgnore и структура проекта

## 1. Установка зависимостей

При разработке проекта может возникнуть потребность добавлять новые таблицы или изменять уже существующие. Ввиду того,
что ORM SQLModel не предусматривает встроенный механизм миграций, необходимо использовать готовые библиотеки 
позволяющие безопасно вносить изменения в таблицы и БД проекта.

Для интеграции Alembic в разрабатываемый проект необходимо его установить через пакетный менеджер:
`pip install alembic`

Сгенерируем папку с миграциями и сопутствующие файлы настроек: `alembic init migrations`

## 2. Настройка

1. В файле alembic.ini переменной sqlalchemy.url указан адрес БД, тот же, что находится в файле connection.py
2. В файле env.py импортированы все из models.py и в переменной target_metadata указано значение target_metadata=SQLModel.metadata
3. В файле script.py.mako импортированы библиотеку sqlmodel
4. Создан файл миграций с помощью команды alembic revision --autogenerate -m "skill added"
5. Применены миграции с помощью команды alembic upgrade head

## 3. Переменные окружения и .gitignore

При разработке приложений часто необходимо хранить ключи и пароли от БД и различного АПИ, к которому обращается сервер.
Для этих целей был создан специальный .gitignore-файл, исключающий индексацию таких файлов при разработке проекта.
Для хранения чувствительных данных был создан файл .env в корне проекта.

Чтобы получить данные из такого файла, был установлен пакет: `pip install python-dotenv`

После установки, в файле connetion.py адрес БД был написан с использованием переменных окружения:

```python
import os
from dotenv import load_dotenv

load_dotenv()
db_url = os.getenv('DB_ADMIN')
```

В 'migrations/env.py' для получения базы данных был написан код:

```python
BASE_DIR = Path(__file__).resolve().parents[1]
load_dotenv(dotenv_path=BASE_DIR / ".env")

db_url = os.getenv("DB_ADMIN")
if not db_url:
    raise RuntimeError("DB_ADMIN is not set (check your .env)")

config.set_main_option("sqlalchemy.url", db_url)
```

# Издательский дом — Клиентская часть

Клиентское приложение для системы управления издательской деятельностью, разработанное с использованием Vue.js 3 и Vuetify 3.

## Технологии

- **Vue.js 3** — прогрессивный JavaScript фреймворк
- **Vuetify 3** — библиотека компонентов Material Design
- **Vue Router 4** — маршрутизация
- **Pinia** — управление состоянием
- **Axios** — HTTP клиент
- **Vite** — сборщик

## Функциональность

### Аутентификация
- Вход в систему
- Регистрация нового пользователя
- Управление профилем
- Смена пароля

### Управление данными (CRUD)
- **Сотрудники** — менеджеры, редакторы и другие
- **Авторы** — каталог авторов книг
- **Книги** — библиотека изданий с авторами и редакторами
- **Контракты** — договоры на издание
- **Заказчики** — база клиентов
- **Заказы** — управление заказами и позициями

### Отчёты
- Список книг по автору
- Ответственные редакторы для всех изданий
- Количество редакторов каждой книги
- Контракты по месяцам за год
- Топ менеджеры по контрактам
- Квартальный отчёт по контрактам

## Установка и запуск

### Предварительные требования
- Node.js 18+
- npm или yarn
- Запущенный backend сервер (Django)

### Установка зависимостей

```bash
cd publishing_frontend
npm install
```

### Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:5173

### Сборка для продакшена

```bash
npm run build
```

## Настройка

### API URL

По умолчанию приложение обращается к API по адресу `http://localhost:8000/api/v1`. 
Для изменения отредактируйте файл `src/services/api.js`.

### CORS

Для работы фронтенда с бэкендом необходимо настроить CORS на стороне Django.
В `settings.py` бэкенда добавлены необходимые настройки:

```python
CORS_ALLOWED_ORIGINS = [
    'http://localhost:5173',
    'http://127.0.0.1:5173',
]
```

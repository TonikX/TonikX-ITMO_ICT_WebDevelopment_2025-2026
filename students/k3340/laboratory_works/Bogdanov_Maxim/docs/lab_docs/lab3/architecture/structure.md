## Описание основных компонентов

### cmd/api/main.go

Точка входа приложения. Инициализирует все зависимости, подключается к базе данных, настраивает HTTP сервер и запускает приложение.

### internal/domain/

Содержит интерфейсы и доменные сущности. Этот слой не зависит от внешних библиотек и определяет контракты для всей системы.

### internal/usecase/

Реализует бизнес-логику приложения. Каждый use case отвечает за определенный сценарий использования.

### internal/infrastructure/

Содержит конкретные реализации интерфейсов из domain слоя. Включает работу с базой данных, JWT, паролями и логированием.

### internal/delivery/http/

HTTP слой приложения. Обрабатывает HTTP запросы, валидирует данные и вызывает соответствующие use cases.

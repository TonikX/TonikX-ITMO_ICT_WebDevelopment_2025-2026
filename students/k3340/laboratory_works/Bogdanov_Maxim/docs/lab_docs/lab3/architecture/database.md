# База данных

## Обзор

База данных PostgreSQL используется для хранения всех данных приложения. Схема БД включает справочные таблицы и основные таблицы для работы системы.

## Основные таблицы

### Справочники

- **academic_years** - Учебные годы
- **subject_types** - Типы дисциплин (базовые/профильные)
- **subjects** - Предметы
- **genders** - Пол
- **weekdays** - Дни недели
- **grading_periods** - Оценочные периоды (четверти)

### Основные таблицы

- **classrooms** - Кабинеты
- **teachers** - Учителя
- **teacher_subjects** - Связь учителей и предметов (многие-ко-многим)
- **classes** - Классы
- **students** - Ученики
- **schedule** - Расписание занятий
- **grades** - Оценки
- **auth_users** - Пользователи системы
- **auth_refresh_tokens** - Refresh токены для JWT

## Миграции

Миграции базы данных управляются через golang-migrate. Все миграции находятся в директории `migrations/`.

### Применение миграций

При запуске через Docker миграции применяются автоматически. Для локальной разработки можно использовать:

```bash
migrate -path migrations -database "postgres://user:pass@localhost/school_db?sslmode=disable" up
```

## Индексы

В базе данных созданы индексы для оптимизации запросов:
- Индексы на внешние ключи
- Индексы для часто используемых полей
- Частичные индексы для активных записей (deleted_at IS NULL)

## Soft Delete

Большинство основных таблиц поддерживают soft delete через поле `deleted_at`. Это позволяет сохранять историю данных и восстанавливать удаленные записи при необходимости.

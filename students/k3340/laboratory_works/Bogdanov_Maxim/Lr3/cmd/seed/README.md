# Скрипт заполнения базы данных

Этот скрипт заполняет базу данных тестовыми данными для разработки и тестирования.

## Что заполняется

### Справочные данные:
- Пол (Мужской, Женский)
- Дни недели (Понедельник - Воскресенье)
- Типы предметов (Гуманитарный, Естественно-научный, Математический, и т.д.)
- Учебные годы (2023-2024, 2024-2025)
- Периоды оценивания (4 четверти для 2024-2025)
- Предметы (16 предметов: Русский язык, Математика, Физика, и т.д.)
- Кабинеты (15 кабинетов)

### Основные данные:
- 10 учителей с закрепленными кабинетами
- Связи учителей и предметов
- 5 классов (9A, 9B, 10A, 10B, 11A) с классными руководителями
- 30 учеников (по 6 в каждом классе)
- Расписание для класса 9A (пример)
- Оценки для нескольких учеников за 1 и 2 четверти
- 4 пользователя для входа:
  - `admin` / `password123` - администратор
  - `teacher1` / `password123` - учитель
  - `teacher2` / `password123` - учитель
  - `headteacher` / `password123` - завуч

## Использование

### Вариант 1: Через Docker (Рекомендуется)

Если база данных запущена через Docker Compose, используйте этот способ:

```bash
# Убедитесь, что БД запущена
docker compose up -d postgres migrate

# Дождитесь готовности БД и завершения миграций
docker compose ps

# Запустите скрипт внутри Docker контейнера
docker run --rm --network lr3_school_network \
  -v ${PWD}:/app -w /app \
  -e DB_HOST=postgres \
  -e DB_PORT=5432 \
  -e DB_USER=postgres \
  -e DB_PASSWORD=postgres \
  -e DB_NAME=school_db \
  -e DB_SSLMODE=disable \
  golang:1.25-alpine sh -c "apk add --no-cache git && go mod download && go run ./cmd/seed/main.go"
```

**Для Windows PowerShell:**
```powershell
docker run --rm --network lr3_school_network `
  -v ${PWD}:/app -w /app `
  -e DB_HOST=postgres `
  -e DB_PORT=5432 `
  -e DB_USER=postgres `
  -e DB_PASSWORD=postgres `
  -e DB_NAME=school_db `
  -e DB_SSLMODE=disable `
  golang:1.25-alpine sh -c "apk add --no-cache git && go mod download && go run ./cmd/seed/main.go"
```

### Вариант 2: Локально (если нет локального PostgreSQL на порту 5432)

```bash
# Убедитесь, что БД запущена в Docker
docker compose up -d postgres migrate

# Запустите скрипт локально
go run ./cmd/seed/main.go
```

**Внимание:** Если на вашем компьютере установлен локальный PostgreSQL на порту 5432, скрипт может подключиться к нему вместо Docker контейнера. В этом случае используйте Вариант 1.

### Вариант 3: Скомпилированный бинарник

```bash
# Скомпилируйте скрипт
go build -o seed ./cmd/seed/main.go

# Запустите (только если нет локального PostgreSQL на порту 5432)
./seed
```

## Параметры подключения

Скрипт использует переменные окружения для подключения к БД:

- `DB_HOST` - хост БД (по умолчанию: `localhost`)
- `DB_PORT` - порт БД (по умолчанию: `5432`)
- `DB_USER` - пользователь БД (по умолчанию: `postgres`)
- `DB_PASSWORD` - пароль БД (по умолчанию: `postgres`)
- `DB_NAME` - имя БД (по умолчанию: `school_db`)
- `DB_SSLMODE` - режим SSL (по умолчанию: `disable`)

## Безопасность

Скрипт использует `ON CONFLICT DO NOTHING` для всех INSERT операций, поэтому его можно запускать многократно без ошибок. Существующие данные не будут перезаписаны.

## Примечания

- Скрипт заполняет данные для текущего учебного года (2024-2025)
- Все пароли пользователей: `password123`
- Скрипт идемпотентен - можно запускать несколько раз

## Архитектурные решения

### Разделение ответственности

Проект следует принципу разделения ответственности:

- **models.py** - структура данных и валидация на уровне модели
- **services.py** - бизнес-логика и сложные запросы
- **forms.py** - валидация пользовательского ввода
- **views.py** - обработка запросов и связь компонентов
- **templates/** - представление данных

Такой подход обеспечивает:
- Легкость тестирования отдельных компонентов
- Переиспользование логики
- Упрощение поддержки и масштабирования

## Сложности и их решение

### 1. Проверка пересечений интервалов

**Проблема:** Необходимо эффективно проверять, не пересекается ли новое бронирование с существующими.

**Решение:** Использована логика отрицания непересечения:
```python
~Q(check_out__lte=check_in) & ~Q(check_in__gte=check_out)
```

Это проще и понятнее, чем прямая проверка всех случаев пересечения.

### 2. Права на оставление отзывов

**Проблема:** Нужно проверить, что пользователь действительно проживал в номере.

**Решение:** 
- Привязка отзыва к конкретному бронированию
- Проверка статуса брони (checked_in/checked_out)
- Валидация даты (можно оставить после заезда)
- Запрет дублирования отзывов на одну бронь

### 3. Редактирование бронирований

**Проблема:** Нельзя разрешить редактирование после заезда или для определённых статусов.

**Решение:** Использование свойств модели `can_edit` и `can_cancel`:
```python
@property
def can_edit(self):
    return (
        date.today() < self.check_in and
        self.status in ['pending', 'confirmed']
    )
```

### 4. Фильтрация с сохранением GET-параметров

**Проблема:** При пагинации терялись параметры фильтров.

**Решение:** В шаблоне добавлен код для сохранения всех GET-параметров:
```html
?page={{ num }}{% for key, value in request.GET.items %}{% if key != 'page' %}&{{ key }}={{ value }}{% endif %}{% endfor %}
```

### 5. Тестирование с датами в прошлом

**Проблема:** Валидация запрещает создавать бронирования с датами в прошлом, но для тестов это нужно.

**Решение:** Добавлен параметр `skip_validation` в метод `save()`:
```python
booking.save(skip_validation=True)
```

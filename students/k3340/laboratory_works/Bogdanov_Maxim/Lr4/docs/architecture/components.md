# Основные компоненты

## API Client (src/api/client.js)

Настройка Axios для работы с REST API.

### Функциональность
- Настройка базового URL для всех запросов
- Добавление токена авторизации в заголовки каждого запроса
- Автоматическое обновление токенов при 401 ошибке
- Обработка ошибок и перенаправление на страницу входа при необходимости

### Особенности реализации
- Использование interceptors для автоматической обработки токенов
- Очередь запросов при обновлении токена для предотвращения множественных запросов

## Store: Auth (src/stores/auth.js)

Управление состоянием аутентификации.

### Функциональность
- Хранение информации о текущем пользователе
- Методы для входа (`login`)
- Методы для регистрации (`register`)
- Методы для выхода (`logout`)
- Проверка аутентификации (`isAuthenticated`)

### Хранимые данные
- Информация о пользователе
- Токены доступа и обновления

## Store: Reference (src/stores/reference.js)

Управление справочными данными.

### Функциональность
- Загрузка справочных данных:
  - Классы
  - Предметы
  - Кабинеты
  - Учебные годы
  - Периоды оценивания
  - Дни недели
- Вспомогательные функции для получения названий по ID:
  - `getClassLabel(classId)`
  - `getTeacherLabel(teacherId)`
  - `getStudentLabel(studentId)`
  - `getSubjectLabel(subjectId)`
  - и другие

### Особенности
- Ленивая загрузка данных (только при необходимости)
- Кэширование загруженных данных

## Router (src/router/index.js)

Конфигурация маршрутизации приложения.

### Маршруты
- `/login` - страница входа
- `/register` - страница регистрации
- `/home` - главная страница
- `/teachers` - управление учителями
- `/students` - управление учениками
- `/classes` - управление классами
- `/schedules` - управление расписанием
- `/grades` - управление оценками
- `/info` - статистика
- `/reports` - отчеты

### Защита маршрутов
Все маршруты, кроме `/login` и `/register`, защищены навигационными хуками. Неавторизованные пользователи перенаправляются на страницу входа.

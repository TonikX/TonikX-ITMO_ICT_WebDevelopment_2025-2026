# Особенности реализации

## Обработка токенов

Реализован механизм автоматического обновления токенов доступа при их истечении.

### Алгоритм работы
1. При получении 401 ошибки выполняется запрос на обновление токена
2. Новые токены сохраняются в localStorage
3. Исходный запрос повторяется с новым токеном
4. При неудачном обновлении пользователь перенаправляется на страницу входа

### Предотвращение множественных запросов
Используется флаг `isRefreshing` и очередь запросов для предотвращения одновременных запросов на обновление токена.

## Отображение данных

Вместо ID отображаются понятные названия для улучшения пользовательского опыта.

### Примеры преобразований
- **Классы**: вместо `ID: 1` отображается `10А`
- **Учителя**: вместо `ID: 5` отображается `Иванов Иван Иванович`
- **Предметы**: вместо `ID: 3` отображается `Математика`
- **Кабинеты**: вместо `ID: 10` отображается `Кабинет 101`

### Реализация
Используются вспомогательные функции из `reference.js` store, которые находят объект по ID и возвращают его читаемое представление.

## Фильтрация и навигация

### Фильтрация учеников
- Выбор класса в верхней части страницы
- Автоматическое обновление списка при изменении фильтра
- Сохранение выбранного класса в URL для возможности прямых ссылок

### Фильтрация расписания
- Аналогичная фильтрация по классу
- Отображение расписания только для выбранного класса

### Навигация между разделами
- Клик на название класса в разделе "Классы" открывает раздел "Ученики" с предустановленным фильтром
- Использование query параметров для передачи данных между страницами

## Формы редактирования

### Автоматическое заполнение
При открытии формы редактирования все поля автоматически заполняются текущими данными объекта.

### Реализация
- Использование `watch` для отслеживания открытия диалога
- Копирование данных объекта для избежания проблем с реактивностью
- Использование `nextTick` для корректного заполнения после рендеринга

## Расписание

### Структура отображения
Расписание отображается в виде отдельных таблиц для каждого дня недели, отсортированных по порядку дня.

### Преимущества
- Более читаемое представление
- Удобная навигация по дням недели
- Легче найти нужный урок

### Реализация
- Группировка записей по `weekday_id`
- Сортировка по `day_order`
- Отдельная таблица для каждого дня

# Структура проекта

## Обзор

Проект организован по стандартной структуре Vue.js приложения с использованием Vite в качестве инструмента сборки.

## Структура директорий

```
Task2/
├── src/                    # Исходный код приложения
│   ├── components/         # Переиспользуемые компоненты
│   │   ├── Hello.vue
│   │   ├── WarriorList.vue
│   │   └── WarriorForm.vue
│   ├── views/             # Страницы приложения
│   │   └── Warriors.vue
│   ├── router/            # Конфигурация роутера
│   │   └── index.js
│   ├── App.vue            # Корневой компонент
│   └── main.js            # Точка входа приложения
├── docs/                  # Документация MkDocs
│   ├── index.md
│   ├── installation.md
│   ├── structure.md
│   ├── components.md
│   └── routing.md
├── index.html             # HTML шаблон
├── vite.config.js         # Конфигурация Vite
├── package.json           # Зависимости и скрипты
├── mkdocs.yml            # Конфигурация MkDocs
└── README.md             # Описание проекта
```

## Описание файлов

### src/main.js

Точка входа приложения. Здесь происходит:
- Импорт и создание экземпляра Vue приложения
- Подключение роутера
- Монтирование приложения в DOM

```javascript
import { createApp } from 'vue'
import App from './App.vue'
import router from './router'

createApp(App).use(router).mount('#app')
```

### src/App.vue

Корневой компонент приложения. Содержит `<router-view/>` для отображения компонентов в зависимости от текущего маршрута.

### src/router/index.js

Конфигурация Vue Router:
- Определение маршрутов
- Создание экземпляра роутера с использованием `createRouter` и `createWebHistory`

### src/components/

Директория для переиспользуемых компонентов:
- **Hello.vue** — простой компонент приветствия
- **WarriorList.vue** — компонент для отображения списка воинов
- **WarriorForm.vue** — форма создания нового воина

### src/views/

Директория для страниц приложения:
- **Warriors.vue** — страница со списком воинов и формой создания

## Принципы организации

1. **Разделение компонентов и views**: Компоненты в `components/`, страницы в `views/`
2. **Модульность**: Каждый компонент в отдельном файле
3. **Options API**: Все компоненты используют Options API (не Composition API)
4. **Однонаправленный поток данных**: Данные передаются через props, события через emit

# Компоненты

## Обзор

Приложение состоит из четырех основных компонентов, каждый из которых выполняет определенную функцию.

## Hello.vue

Простой компонент приветствия, демонстрирующий базовую структуру Vue компонента.

### Структура

```vue
<template>
  <div>
    <h1>Привет, это твой первый компонент</h1>
  </div>
</template>
```

### Использование

Компонент отображается на маршруте `/hi`.

## Warriors.vue

Главная страница приложения, объединяющая функционал работы со списком воинов.

### Функционал

- Получение списка воинов из API
- Отображение формы создания воина
- Отображение списка воинов

### Данные

- `warriors` (Array) — массив воинов

### Методы

- `fetchWarriors()` — получение списка воинов через GET запрос к API

### Жизненный цикл

- `mounted()` — автоматический вызов `fetchWarriors()` при монтировании компонента

### Использование

Компонент отображается на маршруте `/warriors`.

## WarriorList.vue

Компонент для отображения списка воинов.

### Props

- `warriors` (Array, required) — массив объектов воинов

### Структура данных воина

```javascript
{
  id: Number,
  name: String,
  race: String
}
```

### Отображение

Для каждого воина отображается:
- Имя (name)
- Раса (race)

### Использование

```vue
<WarriorList :warriors="warriors" />
```

## WarriorForm.vue

Форма для создания нового воина.

### Данные формы

- `formData.name` (String) — имя воина
- `formData.race` (String) — раса воина

### Методы

- `createWarrior()` — отправка POST запроса на создание воина

### API Endpoint

- `POST http://62.109.28.95:8890/warrior/create1`

### События

После успешного создания воина компонент эмитит событие `warrior-created`, которое обрабатывается родительским компонентом для обновления списка.

### Использование

```vue
<WarriorForm @warrior-created="fetchWarriors" />
```

## Взаимодействие компонентов

```
Warriors.vue (родительский компонент)
├── WarriorForm.vue (форма создания)
│   └── Эмитит событие warrior-created
└── WarriorList.vue (список воинов)
    └── Получает данные через props
```

## Стилизация

Все компоненты имеют собственные стили, определенные в секции `<style scoped>`. Использование `scoped` гарантирует, что стили применяются только к текущему компоненту.

    Структура проекта

    Обзор

    Проект следует стандартной структуре Vue.js приложения с использованием Vite и организован для удобной разработки и поддержки.

    ResidentalConnectFront/
├── docs/                 # Документация MkDocs
├── src/                  # Исходный код приложения
│   ├── assets/          # Статические ресурсы
│   ├── components/      # Переиспользуемые компоненты
│   ├── plugins/         # Плагины (Vuetify)
│   ├── router/          # Конфигурация роутера
│   ├── services/        # API сервисы
│   ├── stores/          # Pinia stores
│   ├── utils/           # Утилиты
│   ├── views/           # Страницы приложения
│   ├── App.vue          # Корневой компонент
│   └── main.js          # Точка входа
├── index.html           # HTML шаблон
├── package.json         # Зависимости и скрипты
├── vite.config.js       # Конфигурация Vite
├── mkdocs.yml          # Конфигурация MkDocs
└── README.md           # Описание проекта

    Детальное описание

    /src

    Основная директория с исходным кодом приложения.

    /src/main.js

    Точка входа приложения. Здесь происходит: - Создание Vue приложения - Подключение плагинов (Pinia, Router, Vuetify) - Монтирование приложения в DOM

    import { createApp } from 'vue'
import { createPinia } from 'pinia'
import App from './App.vue'
import router from './router'
import vuetify from './plugins/vuetify'

const app = createApp(App)
app.use(createPinia())
app.use(router)
app.use(vuetify)
app.mount('#app')

    /src/App.vue

    Корневой компонент приложения. Содержит: - Навигационную панель - Роутер для отображения страниц - Общую структуру приложения

    /src/assets/

    Статические ресурсы: - main.css - глобальные стили

    /src/components/

    Переиспользуемые компоненты Vue. В данный момент пуста, но здесь можно размещать компоненты, используемые на нескольких страницах.

    /src/plugins/

    Плагины Vue: - vuetify.js - конфигурация Vuetify (темы, иконки)

    /src/router/

    Конфигурация Vue Router: - index.js - определение маршрутов и защита доступа

    /src/services/

    Сервисы для работы с API: - api.js - базовый Axios клиент с interceptors - authService.js - методы для аутентификации

    /src/stores/

    Pinia stores для управления состоянием: - auth.js - состояние аутентификации и методы работы с пользователем

    /src/utils/

    Вспомогательные функции и утилиты.

    /src/views/

    Страницы приложения (views): - Home.vue - главная страница - Login.vue - страница входа - Register.vue - страница регистрации - Profile.vue - страница профиля пользователя

    Конфигурационные файлы

    package.json

    Определяет: - Зависимости проекта - Скрипты для запуска и сборки - Метаданные проекта

    vite.config.js

    Конфигурация Vite: - Плагины (Vue, Vuetify) - Алиасы путей (@ для src/) - Настройки сервера разработки - Прокси для API запросов

    index.html

    HTML шаблон приложения. Содержит: - Подключение шрифтов Material Design Icons - Контейнер для Vue приложения (#app)

    Принципы организации

    Разделение по функциональности

    • Views - полные страницы приложения
    • Components - переиспользуемые UI компоненты
    • Services - бизнес-логика и API вызовы
    • Stores - глобальное состояние приложения

    Именование файлов

    • Компоненты: PascalCase.vue (например, Login.vue)
    • Сервисы: camelCase.js (например, authService.js)
    • Stores: camelCase.js (например, auth.js)

    Импорты

    Используется алиас @ для импорта из src/:

    import { useAuthStore } from '@/stores/auth'
import api from '@/services/api'

    Расширение структуры

    При добавлении нового функционала:

    1. Новая страница → создайте в src/views/
    2. Новый компонент → создайте в src/components/
    3. Новый API эндпоинт → добавьте метод в соответствующий сервис
    4. Новое состояние → создайте store в src/stores/

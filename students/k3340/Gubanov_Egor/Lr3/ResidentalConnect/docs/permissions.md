# Ролевая модель и права доступа

## Обзор

Система использует ролевую модель с тремя основными ролями:
- **Жилец** (resident)
- **Мастер** (master)
- **Диспетчер** (dispatcher)

Каждая роль имеет свои права доступа к различным ресурсам.

## Роли пользователей

### Жилец (resident)

**Права доступа:**
- Просмотр информации о своих квартирах
- Создание заявок на обслуживание для своих квартир
- Просмотр своих заявок
- Создание и просмотр показаний счетчиков для своих квартир
- Просмотр категорий услуг

**Ограничения:**
- Не может видеть чужие квартиры
- Не может видеть чужие заявки
- Не может изменять статус заявок
- Не может назначать мастеров

### Мастер (master)

**Права доступа:**
- Просмотр заявок, назначенных ему
- Изменение статуса назначенных заявок
- Добавление комментариев к назначенным заявкам
- Просмотр категорий услуг

**Ограничения:**
- Не может видеть все заявки
- Не может назначать других мастеров
- Не может создавать заявки
- Не может управлять квартирами и домами

### Диспетчер (dispatcher)

**Права доступа:**
- Полный доступ ко всем ресурсам
- Создание и управление домами
- Создание и управление квартирами
- Просмотр всех заявок
- Назначение мастеров на заявки
- Изменение статуса любых заявок
- Просмотр всех показаний счетчиков

## Кастомные permissions

Система использует следующие кастомные permissions:

### IsResident
Проверяет, что пользователь имеет роль "Жилец".

### IsMaster
Проверяет, что пользователь имеет роль "Мастер".

### IsDispatcher
Проверяет, что пользователь имеет роль "Диспетчер".

### IsOwnerOrDispatcher
Проверяет, что пользователь является владельцем объекта (квартиры) или диспетчером.

### IsAssignedWorkerOrDispatcher
Проверяет, что пользователь является назначенным мастером на заявку или диспетчером.

### IsRequesterOrWorkerOrDispatcher
Проверяет, что пользователь является создателем заявки, назначенным мастером или диспетчером.

## Примеры использования

### Жилец создает заявку
```python
# POST /api/service-requests/
# Authorization: Token <token_жильца>
{
  "title": "Протечка крана",
  "description": "Капает кран в ванной",
  "category_id": 1,
  "apartment_id": 5  # Должна быть квартира жильца
}
```

### Мастер изменяет статус заявки
```python
# POST /api/service-requests/1/change_status/
# Authorization: Token <token_мастера>
{
  "status": "done"
}
```

### Диспетчер назначает мастера
```python
# POST /api/service-requests/1/assign_worker/
# Authorization: Token <token_диспетчера>
{
  "worker_id": 3
}
```

## Ошибки доступа

### 403 Forbidden
Пользователь не имеет прав на выполнение действия.

```json
{
  "detail": "У вас нет прав для выполнения данного действия."
}
```

### 404 Not Found
Ресурс не найден или пользователь не имеет к нему доступа (например, жилец пытается получить чужую квартиру).

```json
{
  "detail": "Не найдено."
}
```

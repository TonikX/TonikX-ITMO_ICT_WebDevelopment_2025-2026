# Архитектура приложения

Приложение построено по архитектуре **клиент‑сервер**.

## Backend

Backend реализует REST API, которое предоставляет доступ к следующим
сущностям:

-   Applicants (соискатели)
-   Vacancies (вакансии)
-   Employers (работодатели)
-   Professions (профессии)
-   Education levels (уровни образования)
-   Аналитические запросы

Маршруты API регистрируются с использованием Router Django REST
Framework.

``` python
from rest_framework.routers import DefaultRouter
from exchange.views import ApplicantViewSet, VacancyViewSet

router = DefaultRouter()
router.register(r'applicants', ApplicantViewSet)
router.register(r'vacancies', VacancyViewSet)

urlpatterns = router.urls
```

Каждый ViewSet предоставляет стандартные операции:

-   получение списка объектов
-   получение одного объекта
-   создание
-   изменение
-   удаление

# Клиентская часть (Frontend)

Клиентское приложение создано на Vue 3.

Приложение имеет несколько страниц, между которыми осуществляется
переход с помощью Vue Router.

## Настройка маршрутов

``` javascript
import { createRouter, createWebHistory } from 'vue-router'
import VacanciesView from '../views/VacanciesView.vue'
import ApplicantsView from '../views/ApplicantsView.vue'
import AnalyticsView from '../views/AnalyticsView.vue'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    { path: '/', redirect: '/vacancies' },
    { path: '/vacancies', component: VacanciesView },
    { path: '/applicants', component: ApplicantsView },
    { path: '/analytics', component: AnalyticsView }
  ]
})

export default router
```

Vue Router позволяет создавать SPA‑приложение (Single Page Application),
в котором переход между страницами происходит без перезагрузки браузера.

# Страница Vacancies

На странице Vacancies отображается список вакансий.

Данные загружаются через REST API:

    GET /api/vacancies/

Пример загрузки данных:

``` javascript
async function loadVacancies() {
  const response = await fetch("http://127.0.0.1:8000/api/vacancies/")
  const data = await response.json()
  vacancies.value = data
}
```

Отображение данных реализовано с помощью таблицы Vuetify:

``` vue
<v-data-table
  :headers="headers"
  :items="vacancies"
  items-per-page="10"
></v-data-table>
```

Таблица поддерживает:

-   пагинацию
-   сортировку
-   отображение большого количества записей

# Страница Analytics

Страница Analytics предназначена для выполнения аналитических запросов.

Система позволяет выполнять следующие аналитические запросы:

1.  Профессии соискателей, отсутствующие в вакансиях
2.  Возможные вакансии для соискателей
3.  Количество дней с момента публикации вакансий
4.  Количество активных пособий
5.  Количество вакансий с определённым уровнем образования

Пример запроса:

``` javascript
async function loadAnalytics() {
  const response = await fetch(
    "http://127.0.0.1:8000/api/analytics/applicant-professions-not-in-vacancies/"
  )

  const data = await response.json()
  analyticsResult.value = data
}
```

Результаты отображаются в таблице.

# Реализация авторизации

В приложении реализованы интерфейсы:

-   регистрация пользователя
-   авторизация пользователя
-   изменение учетных данных

Аутентификация реализована через **Token Authentication** Django REST
Framework.

Пример запроса авторизации:

``` javascript
async function login(username, password) {
  const response = await fetch("http://127.0.0.1:8000/api/auth/login/", {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
    },
    body: JSON.stringify({ username, password })
  })

  const data = await response.json()

  localStorage.setItem("token", data.token)
}
```

После авторизации токен сохраняется в localStorage и используется при
последующих запросах к API.

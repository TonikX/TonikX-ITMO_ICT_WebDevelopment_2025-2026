# Лабораторная работа №3: Django REST Framework

**Система управления читальным залом**

## Описание проекта

Полнофункциональная система для управления читальным залом библиотеки, реализованная с использованием Django и Django REST Framework. Система обеспечивает управление читальными залами, читателями, бронированиями, библиотекарями и их расписанием.

## Цель работы

Овладеть практическими навыками и умениями реализации web-сервисов средствами Django и Django REST Framework.

## Вариант задания

**Предметная область:** Читальный зал

**Функциональность:**
- Управление читальными залами (малые, средние, большие)
- Регистрация и управление читателями
- Система бронирований с отслеживанием времени
- Управление библиотекарями и их расписанием
- Аналитика и отчетность по кварталам
- Поиск свободных залов на заданное время

## Технологический стек

### Backend
- **Python** 3.10+
- **Django** 5.1.1 - веб-фреймворк
- **Django REST Framework** 3.15.2 - построение RESTful API
- **Djoser** 2.2.3 - аутентификация и управление пользователями
- **drf-spectacular** 0.27.2 - OpenAPI/Swagger документация
- **SQLite** - база данных (можно заменить на PostgreSQL)

### Дополнительные библиотеки
- **django-cors-headers** - поддержка CORS для фронтенда
- **djangorestframework-simplejwt** - JWT аутентификация

## Архитектура API

### RESTful принципы
- Использование HTTP методов (GET, POST, PUT, DELETE)
- Статус коды ответов (200, 201, 400, 404, 403)
- JSON формат данных
- Версионирование API
- Фильтрация, пагинация, сортировка

### Endpoints структура
```
/api/
├── auth/                  # Аутентификация (Djoser)
│   ├── users/            # Регистрация
│   ├── jwt/              # JWT токены
│   └── users/me/         # Текущий пользователь
├── reading-rooms/         # Читальные залы
│   ├── /                 # Список/создание
│   ├── /{id}/            # Детали/обновление/удаление
│   ├── free/             # Свободные залы
│   └── /{id}/readers/    # Читатели зала
├── readers/              # Читатели
│   ├── /                 # Список/создание
│   ├── /{id}/            # Детали/обновление/удаление
│   ├── count-by-phone/   # Подсчет по телефону
│   ├── /{id}/librarian/  # Библиотекарь читателя
│   └── /{id}/co-readers/ # Совместные бронирования
├── reservations/         # Бронирования
│   ├── /                 # Список/создание
│   └── /{id}/            # Детали/обновление/удаление
├── librarians/           # Библиотекари
│   ├── /                 # Список/создание
│   ├── /{id}/            # Детали/обновление/удаление
│   ├── /{id}/fire/       # Увольнение
│   └── /{id}/hire/       # Прием на работу
├── schedules/            # Расписание
│   ├── /                 # Список/создание
│   └── /{id}/            # Детали/обновление/удаление
└── reports/              # Отчеты
    └── quarter/          # Квартальный отчет
```

## Модель данных

### Основные сущности

**ReadingRoom** (Читальный зал)
- Номер зала
- Этаж
- Тип (малый/средний/большой)
- Вместимость
- Цена за час
- Описание

**Reader** (Читатель)
- Читательский билет (уникальный)
- ФИО
- Телефон
- Email

**Reservation** (Бронирование)
- Читатель (FK)
- Читальный зал (FK)
- Время начала
- Время окончания
- Статус активности

**Librarian** (Библиотекарь)
- ФИО
- Статус активности

**Schedule** (Расписание)
- Библиотекарь (FK)
- День недели
- Этаж

## Права доступа

### После регистрации
Все авторизованные пользователи имеют полный доступ к:
- Управлению читальными залами
- Управлению читателями
- Управлению бронированиями
- Управлению библиотекарями и расписанием
- Просмотру отчетов

### Django Admin Panel
Для доступа к административной панели требуются права `is_staff`.

Используйте скрипт для предоставления прав:
```bash
python make_staff.py
```

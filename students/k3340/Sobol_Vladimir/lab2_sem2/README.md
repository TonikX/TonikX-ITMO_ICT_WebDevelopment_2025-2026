# Лабораторная работа №2 (2 семестр) — Потоки, процессы, асинхронность

## Структура

- `task1/` — параллельное суммирование `1..N` тремя способами (`threading`, `multiprocessing`, `asyncio`).
- `task2/` — параллельный парсер веб-страниц (тот же набор подходов) с сохранением заголовков в SQLite через SQLModel.

## Запуск

```bash
pip install -r requirements.txt

# Task 1 — CPU-bound sum
python task1/benchmark.py

# Task 2 — I/O-bound parser
python task2/benchmark.py
```

## Замечание по Task 1

ТЗ предлагает брать `N = 10**13`. Чистый Python-цикл такой длины занял бы десятки минут и не даёт содержательного сравнения подходов, поэтому в коде выбран `N = 10**8` — этого достаточно, чтобы увидеть реальный эффект GIL и разницу между `threading` и `multiprocessing`. Формулу Гаусса намеренно не используем: задача учебная и требует именно «параллельного» суммирования.

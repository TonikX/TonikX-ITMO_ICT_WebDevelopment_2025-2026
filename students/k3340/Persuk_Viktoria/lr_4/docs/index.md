# Vue.js Frontend - Документация

## Введение

Этот проект представляет собой полнофункциональный frontend на Vue.js 3 для работы с Django REST Framework бэкендом, использующим Djoser для аутентификации через JWT токены.

## Технологический стек

- **Vue 3** - прогрессивный JavaScript фреймворк с Composition API
- **Vue Router** - официальный маршрутизатор для Vue.js
- **Pinia** - современная библиотека управления состоянием для Vue
- **Axios** - HTTP клиент для работы с API
- **Vuetify 3** - Material Design компонентная библиотека
- **Vite** - быстрый сборщик и dev-сервер

## Основные возможности

### Аутентификация и авторизация

- Регистрация новых пользователей
- Вход в систему по email и паролю
- Автоматическое управление JWT токенами (access и refresh)
- Автоматическое обновление токенов при истечении
- Защита маршрутов для авторизованных пользователей

### Управление профилем

- Просмотр профиля пользователя
- Редактирование профиля (display_name, bio, avatar)
- Загрузка аватара
- Просмотр информации о пользователе

### Настройки аккаунта

- Изменение email
- Изменение username
- Изменение пароля

## Архитектура проекта

Проект следует лучшим практикам Vue.js разработки:

- **Composition API** - для всех компонентов
- **Модульная структура** - разделение на API, stores, views, router
- **TypeScript-ready** - структура готова для миграции на TypeScript
- **Адаптивный дизайн** - поддержка мобильных устройств

## Быстрый старт

### Установка зависимостей

```bash
npm install
```

### Запуск в режиме разработки

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173`

### Сборка для продакшена

```bash
npm run build
```

## Настройка

### Переменные окружения

Создайте файл `.env` в корне проекта:

```env
VITE_API_BASE_URL=http://localhost:8000
```

### Настройка CORS на бэкенде

Убедитесь, что Django бэкенд настроен для работы с frontend:

```python
CORS_ALLOWED_ORIGINS = [
    "http://localhost:5173",
    "http://127.0.0.1:5173",
]
```

## API Endpoints

Приложение взаимодействует со следующими endpoints бэкенда:

### Аутентификация

- `POST /auth/jwt/create/` - вход пользователя
- `POST /auth/users/` - регистрация нового пользователя
- `POST /auth/jwt/refresh/` - обновление JWT токена
- `GET /auth/users/me/` - получение данных текущего пользователя
- `PATCH /auth/users/me/` - обновление данных пользователя
- `POST /auth/users/set_password/` - изменение пароля

### Профиль

- `GET /api/profile/` - получение профиля текущего пользователя
- `PATCH /api/profile/` - обновление профиля
- `GET /api/users/<id>/profile/` - получение профиля другого пользователя

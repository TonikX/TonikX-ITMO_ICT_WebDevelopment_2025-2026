# Компоненты

## Обзор

Все компоненты приложения используют Vue 3 Composition API и Vuetify 3 для UI.

## Структура компонентов

### App.vue

Главный компонент приложения, содержит:

- **App Bar** - верхняя панель навигации
  - Логотип/название приложения
  - Кнопки входа/регистрации (для неавторизованных)
  - Меню пользователя (для авторизованных)
- **Navigation Drawer** - боковое меню для мобильных устройств
- **Main Container** - контейнер для контента страниц
- **Snackbar** - глобальные уведомления

**Особенности:**
- Адаптивный дизайн (скрывает/показывает drawer на мобильных)
- Предоставляет функцию `showSnackbar` через `provide/inject`
- Обработка выхода пользователя

### LoginView.vue

Страница входа в систему.

**Поля формы:**
- Email или имя пользователя (обязательное)
- Пароль (обязательное)

**Функциональность:**
- Валидация формы через Vuetify rules
- Интеграция с `authStore.login()`
- Обработка ошибок с отображением в Snackbar
- Редирект на `/profile` или сохраненный `redirect` параметр
- Ссылка на страницу регистрации

**Валидация:**
```javascript
usernameOrEmailRules: [
  (v) => !!v || 'Email или имя пользователя обязательно',
]

passwordRules: [
  (v) => !!v || 'Пароль обязателен',
]
```

### RegisterView.vue

Страница регистрации нового пользователя.

**Поля формы:**
- Email (обязательное, валидация email)
- Username (опциональное, генерируется из email если не указано)
- Пароль (обязательное, минимум 8 символов)
- Подтверждение пароля (обязательное, должно совпадать с паролем)

**Функциональность:**
- Валидация всех полей
- Интеграция с `authStore.register()`
- Автоматический вход после успешной регистрации
- Обработка ошибок с детальными сообщениями
- Редирект на `/profile` после успешной регистрации
- Ссылка на страницу входа

**Валидация:**
```javascript
rePasswordRules: [
  (v) => !!v || 'Подтверждение пароля обязательно',
  (v) => v === password.value || 'Пароли не совпадают',
]
```

### ProfileView.vue

Страница просмотра и редактирования профиля.

**Режим просмотра:**
- Отображение аватара (или иконки по умолчанию)
- Отображаемое имя (display_name)
- Username
- Email
- Дата регистрации
- Биография (bio)
- Кнопка "Редактировать профиль"

**Режим редактирования (модальное окно):**
- Поле для отображаемого имени
- Текстовое поле для биографии
- Загрузка аватара (file input)
- Предпросмотр выбранного аватара
- Кнопки "Отмена" и "Сохранить"

**Функциональность:**
- Загрузка профиля при монтировании компонента
- Редактирование через модальное окно
- Загрузка аватара через FormData
- Обновление профиля через `profileAPI.updateProfile()`
- Обновление store после сохранения
- Форматирование даты регистрации
- Обработка ошибок

**Особенности:**
- Если профиль не существует - отображается информация из user
- Аватар загружается как файл через FormData
- Предпросмотр аватара перед сохранением

### SettingsView.vue

Страница настроек аккаунта.

**Вкладка "Аккаунт":**
- Email (редактируемое)
- Username (редактируемое)
- Кнопка "Сохранить изменения"

**Вкладка "Пароль":**
- Текущий пароль
- Новый пароль (минимум 8 символов)
- Подтверждение нового пароля
- Кнопка "Изменить пароль"

**Функциональность:**
- Использование Vuetify Tabs для переключения между вкладками
- Отдельные формы для каждой вкладки
- Валидация всех полей
- Обновление данных пользователя через `authStore.updateUser()`
- Изменение пароля через `authAPI.changePassword()`
- Очистка формы пароля после успешного изменения
- Обработка ошибок с детальными сообщениями

**Валидация пароля:**
```javascript
newPasswordRules: [
  (v) => !!v || 'Новый пароль обязателен',
  (v) => (v && v.length >= 8) || 'Пароль должен содержать минимум 8 символов',
]

reNewPasswordRules: [
  (v) => !!v || 'Подтверждение пароля обязательно',
  (v) => v === newPassword.value || 'Пароли не совпадают',
]
```

## Общие паттерны

### Использование provide/inject для Snackbar

```vue
<!-- App.vue -->
<script setup>
const showSnackbar = (message, color = 'success') => {
  snackbar.value = { show: true, message, color }
}
provide('showSnackbar', showSnackbar)
</script>

<!-- В дочерних компонентах -->
<script setup>
const showSnackbar = inject('showSnackbar')
showSnackbar('Сообщение', 'success')
</script>
```

### Валидация форм

Все формы используют Vuetify validation:

```vue
<v-form ref="formRef" v-model="valid">
  <v-text-field
    v-model="email"
    :rules="emailRules"
    required
  ></v-text-field>

  <v-btn
    :disabled="!valid"
    @click="handleSubmit"
  >
    Отправить
  </v-btn>
</v-form>

<script setup>
const formRef = ref(null)
const valid = ref(false)

const handleSubmit = async () => {
  const { valid: isValid } = await formRef.value.validate()
  if (!isValid) return
  // Отправка формы
}
</script>
```

### Состояния загрузки

Компоненты показывают состояние загрузки:

```vue
<v-btn
  :loading="authStore.isLoading"
  :disabled="authStore.isLoading"
>
  Отправить
</v-btn>
```

### Обработка ошибок

Единообразная обработка ошибок:

```javascript
try {
  await someAPI()
  showSnackbar('Успех!', 'success')
} catch (error) {
  const message = error.response?.data?.detail ||
                 error.response?.data?.non_field_errors?.[0] ||
                 'Ошибка'
  showSnackbar(message, 'error')
}
```

## Адаптивность

Все компоненты адаптивны благодаря Vuetify Grid System:

```vue
<v-row>
  <v-col cols="12" md="6">
    <!-- Контент -->
  </v-col>
</v-row>
```

- На мобильных: `cols="12"` (полная ширина)
- На планшетах и выше: `md="6"` (половина ширины)

## 🎯 Ключевые особенности реализации

### 1. Структура URL-файлов

**Главный urls.py (проект):**
```python
# django_project_ivanov/urls.py
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('project_first_app.urls')),  # ← Импорт URLs приложения
]
```

**urls.py приложения:**
```python
# project_first_app/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('', views.owner_list, name='owner_list'),
    path('owner/<int:owner_id>/', views.owner_detail, name='owner_detail'),  # ← Параметр в URL
]
```

---

### 2. Передача параметра в контроллер

**URL-паттерн:**
```python
path('owner/<int:owner_id>/', views.owner_detail, name='owner_detail')
```

**Контроллер автоматически получает параметр:**
```python
def owner_detail(request, owner_id):  # ← owner_id передается из URL
    try:
        owner = Avtovladelec.objects.get(pk=owner_id)
    except Avtovladelec.DoesNotExist:
        raise Http404("Владелец не найден")
    return render(request, 'owner.html', {'owner': owner})
```

---

### 3. Обработка ошибок

- **Несуществующий владелец:** HTTP 404 с сообщением "Владелец не найден"
- **Неверный формат ID:** Django автоматически возвращает 404
- **Try-except блок** в контроллере для безопасной работы с базой данных

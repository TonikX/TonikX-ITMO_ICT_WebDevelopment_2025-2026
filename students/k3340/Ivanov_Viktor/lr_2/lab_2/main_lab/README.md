# Лабораторная работа №2 - Табло авиарейсов

**Вариант №3:** Табло отображения информации об авиаперелетах

**Автор:** Иванов Виктор  
**Дата:** 15 октября 2025

---

## Описание проекта

Веб-приложение для управления авиарейсами с возможностью:
- Просмотра расписания рейсов
- Регистрации пользователей
- Бронирования мест на рейсах
- Написания отзывов о рейсах
- Управления бронированиями через админ-панель

---

## Выполненные требования

### Согласно заданию реализовано:

**Хранение информации о рейсах:**
- Номер рейса
- Авиакомпания
- Отлет и прилет (время и город)
- Тип рейса (отлет/прилет)
- Номер гейта
- Дополнительно: количество мест, цена

**Регистрация новых пользователей**
- Расширенная модель User (AbstractUser)
- Поля: телефон, паспорт, дата рождения

**Просмотр и резервирование мест на рейсах**
- Список всех рейсов с поиском
- Детальная информация о рейсе
- Бронирование мест (авторизованные пользователи)

**Редактирование и удаление своих резервирований**
- Список «Мои бронирования»
- Отмена бронирования

**Регистрация пассажиров администратором**
- Django Admin с возможностью:
  - Подтверждения бронирований
  - Назначения номера билета
  - Назначения номера места

**Таблица пассажиров рейса**
- На странице рейса отображается список пассажиров с билетами

**Написание отзывов к рейсам**
- Форма отзыва с полями:
  - Дата рейса
  - Текст отзыва
  - Рейтинг (1-10)
  - Информация о комментаторе (автоматически)

## Установка и запуск

### 1. Клонирование/копирование проекта

Проект находится в папке `main_lab/`

### 2. Создание виртуального окружения (если нет)

```bash
cd main_lab
python3 -m venv venv
source venv/bin/activate  # для Mac/Linux
# или
venv\Scripts\activate     # для Windows
```

### 3. Установка зависимостей

```bash
pip install django
```

### 4. Применение миграций (если нужно)

```bash
python manage.py migrate
```

### 5. Создание суперпользователя (если нужно)

```bash
python manage.py createsuperuser
```

### 6. Загрузка тестовых данных (опционально)

```bash
python load_test_data.py
```

### 7. Запуск сервера

```bash
python manage.py runserver
```

Сервер будет доступен по адресу: **http://127.0.0.1:8000/**

## Модели данных

### User (Пользователь)
Расширенная модель AbstractUser:
- Стандартные поля Django: username, first_name, last_name, email, password
- Дополнительные: phone_number, passport_number, date_of_birth

### Flight (Рейс)
- flight_number - номер рейса
- airline - авиакомпания
- departure_city, arrival_city - города
- departure_time, arrival_time - время
- flight_type - тип (отлет/прилет)
- gate_number - номер гейта
- total_seats - всего мест
- price - цена билета

### Reservation (Резервирование)
- user - пользователь (ForeignKey)
- flight - рейс (ForeignKey)
- ticket_number - номер билета
- seat_number - номер места
- status - статус (pending/confirmed/cancelled)
- is_confirmed - подтверждено администратором

### Review (Отзыв)
- user - пользователь (ForeignKey)
- flight - рейс (ForeignKey)
- flight_date - дата рейса
- text - текст отзыва
- rating - рейтинг (1-10)

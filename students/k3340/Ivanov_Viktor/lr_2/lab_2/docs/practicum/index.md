## модели данных

### User (Пользователь-владелец)
расширенная модель AbstractUser с дополнительными полями:

- `username` - логин пользователя (стандартное поле Django)
- `first_name` - имя (стандартное поле Django)
- `last_name` - фамилия (стандартное поле Django)
- `email` - электронная почта (стандартное поле Django)
- `password` - пароль (хешированный, стандартное поле Django)
- `patronymic` - отчество (VARCHAR 30)
- `data_rozhdeniya` - дата рождения (DATE)
- `passport_number` - номер паспорта (VARCHAR 20, добавлено в практикуме №3)
- `home_address` - домашний адрес (TEXT, добавлено в практикуме №3)
- `nationality` - национальность (VARCHAR 50, добавлено в практикуме №3)

### Avtomobil (Автомобиль)
информация об автомобилях:

- `id_avtomobilya` - ID автомобиля (первичный ключ, AUTO_INCREMENT)
- `gos_nomer` - государственный номер (VARCHAR 15)
- `marka` - марка автомобиля (VARCHAR 20)
- `model` - модель автомобиля (VARCHAR 20)
- `cvet` - цвет автомобиля (VARCHAR 30)

### Vladenie (Владение)
связь между владельцами и автомобилями:

- `id_vladeltsa_avto` - ID записи владения (первичный ключ, AUTO_INCREMENT)
- `id_vladelca` - ID владельца (внешний ключ → User)
- `id_avtomobilya` - ID автомобиля (внешний ключ → Avtomobil)
- `data_nachala` - дата начала владения (DATETIME)
- `data_okonchaniya` - дата окончания владения (DATETIME, nullable)

### Voditelskoe_udostoverenie (Водительское удостоверение)
информация о водительских удостоверениях:

- `id_udostovereniya` - ID удостоверения (первичный ключ, AUTO_INCREMENT)
- `id_vladelca` - ID владельца (внешний ключ → User)
- `nomer_udostovereniya` - номер удостоверения (VARCHAR 10)
- `tip` - тип удостоверения (VARCHAR 10)
- `data_vydachi` - дата выдачи (DATETIME)

## диаграмма связей

```
User (владелец)
    ↓ one-to-many
Voditelskoe_udostoverenie (удостоверение)

User (владелец) ← many-to-many → Avtomobil (автомобиль)
    ↓                                  ↑
Vladenie (промежуточная таблица с доп. полями)
```

## функционал

### функциональные представления (владельцы)
реализованы с использованием обычных функций Python:

- `owner_list()` - получение списка всех владельцев из базы данных, передача данных в шаблон, отображение в виде таблицы
- `owner_detail()` - получение конкретного владельца по ID, получение связанных автомобилей и удостоверений, отображение всех данных
- `owner_create()` - обработка GET/POST запросов, валидация формы, сохранение нового пользователя, автоматический вход в систему

### классовые представления (автомобили)
реализованы с использованием встроенных классов Django:

- `AvtomobilListView` (ListView) - отображение списка всех автомобилей, автоматическая пагинация
- `AvtomobilDetailView` (DetailView) - отображение детальной информации, автоматическое получение объекта по ID
- `AvtomobilCreateView` (CreateView) - создание нового автомобиля, автоматическая обработка формы
- `AvtomobilUpdateView` (UpdateView) - редактирование существующего автомобиля, заполнение формы текущими данными
- `AvtomobilDeleteView` (DeleteView) - удаление автомобиля, запрос подтверждения

### формы (ModelForm)
все формы созданы на основе моделей:

- `OwnerRegistrationForm` - регистрация владельца с расширенными полями, хеширование пароля
- `AvtomobilForm` - создание и редактирование автомобилей, валидация госномера

### админ-панель
настроена кастомная админ-панель с расширенными возможностями:

- отображение дополнительных полей в списках
- фильтры по различным параметрам
- поиск по ключевым полям
- inline редактирование связанных объектов
- кастомные методы для отображения

## особенности реализации

**расширение модели пользователя**
использование AbstractUser для добавления дополнительных полей, настройка AUTH_USER_MODEL в settings.py, создание миграций для новой модели, обновление всех связей.

**связи между моделями**
ForeignKey для связи один-ко-многим (User → Voditelskoe_udostoverenie), ManyToMany через промежуточную модель (User ↔ Avtomobil через Vladenie), добавление дополнительных полей в промежуточную таблицу.

**два подхода к представлениям**
функциональные представления - больше контроля, явная обработка, классовые представления - меньше кода, встроенный функционал, переиспользуемость.

**безопасность**
хеширование паролей через Django auth, CSRF защита для всех форм, валидация данных на уровне форм и моделей, защита от SQL-инъекций через ORM.

**пользовательский интерфейс**
адаптивный дизайн для всех устройств, современные CSS стили, понятная навигация, информативные сообщения об ошибках.

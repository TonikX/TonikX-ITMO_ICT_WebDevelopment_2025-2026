## что было сделано

### 1. создание кастомной модели User

**до (стандартная модель User):**
```python
from django.contrib.auth.models import User

# использовалась встроенная модель Django
```

**после (кастомная модель на основе AbstractUser):**
```python
from django.contrib.auth.models import AbstractUser

class User(AbstractUser):
    # базовые поля
    patronymic = models.CharField(max_length=30)
    data_rozhdeniya = models.DateField()
    
    # новые расширенные поля ⭐
    passport_number = models.CharField(max_length=20, blank=True)
    home_address = models.TextField(blank=True)
    nationality = models.CharField(max_length=50, blank=True)
```

### 2. настройка AUTH_USER_MODEL

в файле `settings.py` добавлена настройка:

```python
# указываем Django использовать нашу кастомную модель
AUTH_USER_MODEL = 'project_first_app.User'
```

эта настройка критически важна и должна быть добавлена **до первой миграции** или требует пересоздания базы данных.

## новые поля модели User

### passport_number (номер паспорта)
- **тип:** CharField(max_length=20)
- **обязательное:** нет (blank=True)
- **описание:** номер паспорта владельца автомобиля
- **формат:** произвольный текст, до 20 символов
- **пример:** "4509 123456", "45 09 123456"

### home_address (домашний адрес)
- **тип:** TextField
- **обязательное:** нет (blank=True)
- **описание:** полный домашний адрес владельца
- **формат:** произвольный текст, без ограничений по длине
- **пример:** "г. Москва, ул. Ленина, д. 10, кв. 25"

### nationality (национальность)
- **тип:** CharField(max_length=50)
- **обязательное:** нет (blank=True)
- **описание:** национальность владельца
- **формат:** текст, до 50 символов
- **пример:** "Русский", "Татарин", "Украинец"

**примечание:** все новые поля имеют параметр `blank=True`, что делает их необязательными при регистрации. это позволяет постепенно заполнять профиль пользователя.

## технические детали

### AbstractUser vs User

**AbstractUser:**
- наследует все функции встроенной модели User
- позволяет добавлять дополнительные поля
- сохраняет всю логику аутентификации Django
- требует настройки AUTH_USER_MODEL

**User (встроенная модель):**
- нельзя напрямую добавить поля
- требует создания отдельной модели Profile с OneToOne связью
- может приводить к усложнению кода

### миграция существующего проекта

**проблемы при миграции:**
1. существующие данные привязаны к встроенной модели User
2. ForeignKey связи должны быть обновлены
3. требуется пересоздание базы данных

**решение:**
1. создание новой модели User(AbstractUser)
2. настройка AUTH_USER_MODEL
3. обновление всех ForeignKey на settings.AUTH_USER_MODEL
4. удаление старой базы данных
5. создание новых миграций
6. применение миграций
7. создание нового суперпользователя
8. загрузка тестовых данных

### важные настройки в settings.py

```python
# указание кастомной модели пользователя
AUTH_USER_MODEL = 'project_first_app.User'

# должно быть добавлено ПЕРЕД первой миграцией!
```

### правильное использование в моделях

```python
from django.conf import settings

class SomeModel(models.Model):
    user = models.ForeignKey(
        settings.AUTH_USER_MODEL,  # правильно ✅
        # НЕ 'auth.User' ❌
        on_delete=models.CASCADE
    )
```

- вся информация в одной таблице

### совместимость
- полная совместимость с Django auth

- эффективные JOIN операции

## возможные проблемы и решения

### проблема: миграции не применяются

**причина:** существующая база данных несовместима с новой моделью

**решение:**
```bash
# удалить базу данных
rm db.sqlite3

# удалить старые миграции
rm project_first_app/migrations/0001_initial.py

# создать новые миграции
python manage.py makemigrations

# применить
python manage.py migrate

# создать суперпользователя
python manage.py createsuperuser
```

### проблема: ошибка при обращении к User

**причина:** использование прямой ссылки на модель User

**неправильно:**
```python
from django.contrib.auth.models import User  # ❌
```

**правильно:**
```python
from django.conf import settings
from django.contrib.auth import get_user_model

User = get_user_model()  # ✅
# или
ForeignKey(settings.AUTH_USER_MODEL)  # ✅
```

### проблема: новые поля не видны в админке

**причина:** не настроен CustomUserAdmin

**решение:** добавить кастомный класс админки с fieldsets

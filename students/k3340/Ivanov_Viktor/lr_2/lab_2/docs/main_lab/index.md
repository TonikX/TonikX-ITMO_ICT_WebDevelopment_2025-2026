# основная лабораторная работа: табло авиарейсов

## описание

полнофункциональное веб-приложение для управления информацией об авиарейсах. реализована система регистрации пользователей, бронирования мест на рейсах, написания отзывов и администрирования через Django Admin.

**вариант №3:** Табло отображения информации об авиаперелетах

## требования

Python 3.13+, Django 5.2.7+, SQLite 3 (встроен в Python).

## установка и запуск

### 1. создание виртуального окружения (если нет)

```bash
cd main_lab
python3 -m venv venv
source venv/bin/activate  # для Mac/Linux
# или
venv\Scripts\activate     # для Windows
```

### 2. установка зависимостей

```bash
pip install django
```

### 3. применение миграций (если требуется)

```bash
python manage.py migrate
```

### 4. создание суперпользователя (если требуется)

```bash
python manage.py createsuperuser
```

### 5. загрузка тестовых данных (опционально)

```bash
python load_test_data.py
```

этот скрипт создаст:
- 4 пользователя (включая admin)
- 5 рейсов различных авиакомпаний
- 6 бронирований в различных статусах

### 6. запуск сервера

```bash
python manage.py runserver
```

сервер будет доступен по адресу: http://127.0.0.1:8000/

## модели данных

### User (Пользователь)
расширенная модель AbstractUser для хранения информации о пассажирах:

**стандартные поля Django:**
- `username` - логин пользователя (уникальный)
- `first_name` - имя
- `last_name` - фамилия
- `email` - электронная почта
- `password` - пароль (хешированный)

**дополнительные поля:**
- `phone_number` - номер телефона (VARCHAR 20)
- `passport_number` - номер паспорта (VARCHAR 20)
- `date_of_birth` - дата рождения (DATE)

### Flight (Рейс)
информация об авиарейсах:

- `flight_number` - номер рейса (VARCHAR 10, уникальный)
- `airline` - авиакомпания (VARCHAR 100)
- `departure_city` - город отправления (VARCHAR 100)
- `arrival_city` - город прибытия (VARCHAR 100)
- `departure_time` - время вылета (DATETIME)
- `arrival_time` - время прилета (DATETIME)
- `flight_type` - тип рейса (VARCHAR 10: 'departure' или 'arrival')
- `gate_number` - номер гейта (VARCHAR 10)
- `total_seats` - общее количество мест (INTEGER)
- `price` - цена билета (DECIMAL 10,2)

**вычисляемые поля (аннотации):**
- средний рейтинг из отзывов
- количество доступных мест

### Reservation (Резервирование/Бронирование)
информация о бронированиях пассажиров:

- `user` - пользователь (ForeignKey → User)
- `flight` - рейс (ForeignKey → Flight)
- `ticket_number` - номер билета (VARCHAR 20, уникальный)
- `seat_number` - номер места (VARCHAR 5)
- `status` - статус бронирования (VARCHAR 20: pending/confirmed/cancelled)
- `is_confirmed` - подтверждено администратором (BOOLEAN)
- `created_at` - дата создания бронирования (DATETIME, auto_now_add)

**ограничения:**
- unique_together на (user, flight) - один пользователь может забронировать рейс только один раз

### Review (Отзыв)
отзывы пользователей о рейсах:

- `user` - пользователь (ForeignKey → User)
- `flight` - рейс (ForeignKey → Flight)
- `flight_date` - дата рейса (DATE)
- `text` - текст отзыва (TEXT)
- `rating` - рейтинг от 1 до 10 (INTEGER)
- `created_at` - дата создания отзыва (DATETIME, auto_now_add)

## диаграмма связей

```
User (пользователь)
    ↓ one-to-many
Reservation (бронирование) ← many-to-one → Flight (рейс)

User (пользователь)
    ↓ one-to-many
Review (отзыв) ← many-to-one → Flight (рейс)
```

## доступные URL-адреса

### главная и рейсы

**главная страница с поиском**
- URL: http://127.0.0.1:8000/
- описание: главная страница с формой поиска рейсов
- фильтры: город отправления, город прибытия, дата, тип рейса
- доступ: для всех пользователей

**список всех рейсов**
- URL: http://127.0.0.1:8000/flights/
- описание: полный список доступных рейсов
- отображает: номер рейса, авиакомпания, маршрут, время, цена, доступные места
- доступ: для всех пользователей

**детальная информация о рейсе**
- URL: http://127.0.0.1:8000/flights/<id>/
- пример: http://127.0.0.1:8000/flights/1/
- описание: полная информация о рейсе
- содержит:
  - все данные о рейсе
  - список пассажиров с подтвержденными билетами
  - все отзывы о рейсе с рейтингами
  - кнопку бронирования (для авторизованных пользователей)
- доступ: для всех пользователей

### аутентификация

**регистрация нового пользователя**
- URL: http://127.0.0.1:8000/register/
- описание: форма регистрации с расширенными полями
- поля: логин, пароль, имя, фамилия, email, телефон, паспорт, дата рождения
- автоматический вход после успешной регистрации
- доступ: для неавторизованных пользователей

**вход в систему**
- URL: http://127.0.0.1:8000/login/
- описание: форма входа в систему
- поля: логин, пароль
- перенаправление на главную после входа
- доступ: для неавторизованных пользователей

**выход из системы**
- URL: http://127.0.0.1:8000/logout/
- описание: выход из системы
- перенаправление на главную
- доступ: для авторизованных пользователей

### бронирования (требуется авторизация)

**создать бронирование**
- URL: http://127.0.0.1:8000/flights/<id>/reserve/
- пример: http://127.0.0.1:8000/flights/1/reserve/
- описание: форма бронирования места на рейсе
- валидация:
  - проверка доступности мест
  - проверка отсутствия дубликатов бронирования
- автоматическая генерация номера билета
- доступ: только для авторизованных пользователей

**мои бронирования**
- URL: http://127.0.0.1:8000/my-reservations/
- описание: список всех бронирований текущего пользователя
- отображает: рейс, номер билета, место, статус
- возможность отмены бронирования
- доступ: только для авторизованных пользователей

**отменить бронирование**
- URL: http://127.0.0.1:8000/reservations/<id>/cancel/
- пример: http://127.0.0.1:8000/reservations/1/cancel/
- описание: отмена существующего бронирования
- изменяет статус на 'cancelled'
- подтверждение перед отменой
- доступ: только автор бронирования

### отзывы (требуется авторизация)

**написать отзыв о рейсе**
- URL: http://127.0.0.1:8000/flights/<id>/review/
- пример: http://127.0.0.1:8000/flights/1/review/
- описание: форма создания отзыва о рейсе
- поля: дата рейса, текст отзыва, рейтинг (1-10)
- автоматическое добавление информации о пользователе
- доступ: только для авторизованных пользователей

**мои отзывы**
- URL: http://127.0.0.1:8000/my-reviews/
- описание: список всех отзывов текущего пользователя
- отображает: рейс, дата, рейтинг, текст
- возможность удаления своих отзывов
- доступ: только для авторизованных пользователей

**удалить отзыв**
- URL: http://127.0.0.1:8000/reviews/<id>/delete/
- пример: http://127.0.0.1:8000/reviews/1/delete/
- описание: удаление своего отзыва
- подтверждение перед удалением
- доступ: только автор отзыва

### администрирование

## особенности реализации

### расширение модели пользователя (AbstractUser)

использование AbstractUser вместо стандартной модели User:
- добавлены поля: телефон, паспорт, дата рождения
- настройка AUTH_USER_MODEL в settings.py
- полная интеграция с системой аутентификации Django
- кастомная форма регистрации

### валидация и ограничения

**на уровне моделей:**
- уникальность номеров рейсов
- уникальность пары (пользователь, рейс) для бронирований
- ограничение рейтинга от 1 до 10
- обязательные поля

**на уровне представлений:**
- проверка доступности мест перед бронированием
- проверка отсутствия дубликатов бронирования
- проверка прав доступа к операциям
- валидация форм

### аннотации и агрегация

использование Django ORM для вычисляемых полей:
- средний рейтинг рейса из отзывов
- количество подтвержденных бронирований
- количество доступных мест
- статистика по рейсам

### система сообщений

- информационные сообщения об успешных операциях
- предупреждения о проблемах
- сообщения об ошибках
- красивое отображение в интерфейсе

### безопасность

**реализованные меры:**
- хеширование паролей через Django auth
- CSRF защита для всех форм
- проверка прав доступа к операциям
- защита от SQL-инъекций через ORM
- валидация данных на сервере

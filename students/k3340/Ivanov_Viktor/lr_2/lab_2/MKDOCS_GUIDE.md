# Руководство по работе с MkDocs

## Что такое MkDocs?

MkDocs - это генератор статической документации из файлов Markdown. Он позволяет создавать красивую, современную документацию для проектов.

## Установка

```bash
cd lr_2/lab_2
python3 -m venv venv
source venv/bin/activate  # для Mac/Linux
# или
venv\Scripts\activate     # для Windows
```

### 2. Установка MkDocs и темы Material

```bash
pip install mkdocs
pip install mkdocs-material
```

## Основные команды

### Предварительный просмотр (Development Server)

Запускает локальный сервер с автоматическим обновлением при изменении файлов:

```bash
mkdocs serve
```

После запуска сайт будет доступен по адресу: http://127.0.0.1:8000/

**Преимущества:**
- Автоматическое обновление при изменении файлов
- Мгновенная проверка изменений
- Удобно для разработки

### Сборка статического сайта

Генерирует финальную версию сайта в папке `site/`:

```bash
mkdocs build
```

**Когда использовать:**
- Перед публикацией на GitHub Pages
- Для создания финальной версии
- Для проверки финального результата

### Очистка собранного сайта

Удаляет папку `site/`:

```bash
mkdocs build --clean
```

### Публикация на GitHub Pages

Автоматически собирает и публикует документацию:

```bash
mkdocs gh-deploy
```

**Что происходит:**
1. Собирается статический сайт
2. Создается/обновляется ветка `gh-pages`
3. Сайт публикуется на GitHub Pages

## Редактирование документации

### Создание новой страницы

1. Создайте новый файл `.md` в папке `docs/`
2. Добавьте его в `mkdocs.yml` в секцию `nav:`

**Пример:**

```yaml
nav:
  - главная: index.md
  - новая страница: new_page.md
```

### Форматирование текста

Используйте Markdown синтаксис:

```markdown
# Заголовок 1
## Заголовок 2
### Заголовок 3

**жирный текст**
*курсив*

- список
- элементов

1. нумерованный
2. список

[ссылка](https://example.com)

`код`

```python
# блок кода
print("Hello, World!")
```
```

### Дополнительные возможности Material theme

**Предупреждения:**

```markdown
!!! note "Заметка"
    Текст заметки

!!! warning "Предупреждение"
    Текст предупреждения

!!! danger "Опасность"
    Важное предупреждение
```

**Вкладки:**

```markdown
=== "Python"
    ```python
    print("Hello")
    ```

=== "JavaScript"
    ```javascript
    console.log("Hello")
    ```
```

**Таблицы:**

```markdown
| Колонка 1 | Колонка 2 |
|-----------|-----------|
| Ячейка 1  | Ячейка 2  |
```

## Конфигурация (mkdocs.yml)

### Основные настройки

```yaml
site_name: Название сайта
site_description: Описание
site_author: Автор
site_url: URL сайта

repo_name: Имя репозитория
repo_url: URL репозитория
```

### Настройка темы

```yaml
theme:
  name: material
  palette:
    - scheme: default
      primary: indigo
      accent: indigo
```

### Навигация

```yaml
nav:
  - главная: index.md
  - раздел:
    - подраздел 1: section/page1.md
    - подраздел 2: section/page2.md
  - о проекте: about.md
```

## Полезные советы

### 1. Проверка перед публикацией

Всегда запускайте `mkdocs serve` и проверяйте все страницы перед публикацией.

### 2. Использование изображений

Создайте папку `docs/images/` и размещайте изображения там:

```markdown
![Описание](images/screenshot.png)
```

### 3. Внутренние ссылки

Используйте относительные пути:

```markdown
[Ссылка на другую страницу](other_page.md)
[Ссылка на раздел](folder/page.md)
```

### 4. Автоматическое обновление

При использовании `mkdocs serve` страница автоматически обновляется при сохранении файлов.

### 5. Проверка битых ссылок

MkDocs предупредит о битых ссылках при сборке.

## Возможные проблемы

### Проблема: команда mkdocs не найдена

**Решение:** Убедитесь, что виртуальное окружение активировано и MkDocs установлен:

```bash
source venv/bin/activate
pip install mkdocs mkdocs-material
```

### Проблема: страница не отображается

**Решение:** Проверьте:
1. Файл существует в папке `docs/`
2. Путь в `nav:` корректный
3. Файл имеет расширение `.md`

### Проблема: изменения не отображаются

**Решение:**
1. Проверьте, что файл сохранен
2. Перезапустите `mkdocs serve`
3. Очистите кэш браузера (Ctrl+Shift+R)

### Проблема: ошибка при публикации на GitHub Pages

**Решение:**
1. Проверьте настройки репозитория
2. Убедитесь, что вы имеете права на push
3. Проверьте наличие ошибок в `mkdocs.yml`

### Добавление нового раздела

1. Создайте файл в `docs/` или подпапке
2. Добавьте ссылку в `mkdocs.yml`:

```yaml

  - главная: index.md

  - новый раздел: new_section/index.md
  - о проекте: about.md
```

3. Запустите `mkdocs serve` для проверки

# Лабораторная работа 3.1

Программная система предназначена для автоматизации работы завуча школы. Решение построено на стеке **Django + Django REST Framework + Djoser** и покрывает:

- ведение справочников учителей, кабинетов, учебных предметов и классов;
- хранение сведений об учениках и их четвертных оценках;
- планирование расписания и закрепление преподавателей за классами и предметами;
- аналитические запросы, необходимые завучу;
- генерацию отчёта об успеваемости класса (JSON + PDF);
- документацию API через OpenAPI и MkDocs.

## Запуск проекта

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

После запуска основные точки входа:

- `http://127.0.0.1:8000/admin/` — административная панель Django;
- `http://127.0.0.1:8000/api/schema/swagger-ui/` — интерактивная OpenAPI-документация;
- `http://127.0.0.1:8000/api/` — REST API приложения;
- `mkdocs serve` — локальный просмотр текстовой документации.

## Аутентификация

Регистрация и токенная авторизация реализованы средствами **Djoser** (`/api/auth/`). Все рабочие эндпоинты требуют авторизации по токену (`Authorization: Token <value>`), кроме служебных эндпоинтов Djoser.

## Структура приложения

- `school.models` — ORM-модели домена: учителя, ученики, классы, расписание, оценки.
- `school.serializers` — сериализаторы DRF для CRUD и специализированных запросов.
- `school.views` — viewset'ы и аналитические представления, в том числе отчёты.
- `school.services` — бизнес-логика формирования отчётов и PDF-файлов.
- `docs/` — документация для MkDocs.

Подробные описания эндпоинтов и примеры запросов приведены в разделе «API».***

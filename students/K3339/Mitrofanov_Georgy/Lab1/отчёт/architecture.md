# Архитектура решений

## Общая идея
Каждое задание — отдельная мини‑система (клиент/сервер или только сервер), где обмен данными происходит через сокеты:
- UDP: `sendto/recvfrom`
- TCP: `connect/listen/accept`, чтение/запись через `send/recv`
- HTTP: TCP‑соединение + текстовый протокол HTTP поверх него

---

## Задание 1 — UDP “Hello”
### Сервер
- создаёт UDP‑сокет `AF_INET + SOCK_DGRAM`
- принимает сообщение от клиента (`recvfrom`)
- печатает его
- отправляет ответ обратно на адрес клиента (`sendto`)

### Клиент
- создаёт UDP‑сокет
- отправляет строку `Hello, server`
- ждёт и печатает ответ `Hello, client`

---

## Задание 2 — TCP сервер мат. операций (вариант 4)
### Обмен данными
- Клиент отправляет запрос в формате JSON (одна строка + `\n`)
- Сервер читает данные до `\n`, парсит JSON, считает результат, возвращает JSON‑ответ

### Вариант 4
- Клиент формирует запрос вида:
  - `op = "parallelogram_area"`
  - параметры: `a`, `h`
- Сервер рассчитывает площадь параллелограмма:
  - `S = a * h`

---

## Задание 3 — HTTP сервер с отдачей `index.html`
- Сервер принимает TCP‑соединение
- читает HTTP‑запрос (первая строка и заголовки)
- открывает файл `index.html` и читает содержимое
- формирует HTTP‑ответ:
  - статус `200 OK`
  - заголовки (`Content-Type`, `Content-Length`)
  - тело ответа (HTML)

---

## Задание 4 — многопользовательский чат (TCP + threading)
### Сервер
- принимает подключения `accept()`
- на каждого клиента создаёт поток:
  - читает сообщения от клиента
  - рассылает всем остальным подключённым клиентам
- хранит список клиентов (сокетов) для рассылки

### Клиент
- отдельный файл `client.py` для каждого запуска
- один поток читает ввод пользователя и отправляет на сервер
- второй поток постоянно слушает сервер и печатает входящие сообщения

---

## Задание 5 — HTTP GET/POST + журнал оценок
### Логика
- Сервер принимает HTTP‑запрос
- Если `GET`:
  - формирует HTML‑страницу со списком дисциплин и оценок
- Если `POST`:
  - читает тело запроса (например: `subject=Математика&grade=5`)
  - добавляет оценку в структуру данных
  - сохраняет изменения в JSON (например `journal.json`)
  - возвращает HTML (обычно редирект или та же страница с обновлёнными данными)

### Хранение оценок (структурировано)
Формат:
- ключ — название дисциплины
- значение — список оценок

Пример:
```json
{
  "Математика": [5, 4],
  "Физика": [3]
}
```

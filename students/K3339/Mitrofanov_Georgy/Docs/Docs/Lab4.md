## 1. Структура маршрутов

* `/login` - Страница входа 
* `/register` - Страница регистрации 
* `/` - Главная страница (список файлов и папок)
* `/folder/:id` - Просмотр содержимого папки
* `/profile` - Профиль пользователя 

## 2. Описание интерфейсов

### 2.1 Страница входа (Login)

**URL:** `/login`

**API запрос:** `POST /api/auth/jwt/create/`

Назначение: авторизация пользователя в системе.

Функции:
- ввод имени пользователя
- ввод пароля
- отправка запроса авторизации
- сохранение JWT токена в localStorage
- редирект на главную страницу после успешного входа

---

### 2.2 Страница регистрации (Register)

**URL:** `/register`

**API запрос:** `POST /api/auth/users/`

Назначение: создание нового пользователя.

Функции:
- ввод имени пользователя
- ввод email (опционально)
- ввод пароля
- подтверждение пароля
- автоматический вход после успешной регистрации

---

### 2.3 Главная страница (Files)

**URL:** `/`

**API запросы:**
- `GET /api/files/` — получение списка файлов
- `GET /api/folders/` — получение списка папок
- `POST /api/files/bulk_upload/` — загрузка файлов
- `POST /api/folders/` — создание папки

Назначение: управление файлами и папками пользователя.

Функции:
- просмотр всех файлов и папок в корневой директории
- загрузка новых файлов (один или несколько)
- создание новой папки
- переход в папку по клику
- отображение размера и типа файлов
- кнопки действий для каждого файла

---

### 2.4 Страница папки (Folder)

**URL:** `/folder/:id`

**API запросы:**
- `GET /api/folders/{id}/content/` — содержимое папки
- `GET /api/folders/{id}/` — информация о папке

Назначение: просмотр содержимого конкретной папки.

Функции:
- просмотр файлов и вложенных папок
- навигация "назад" к родительской папке
- отображение пути к текущей папке
- загрузка файлов в текущую папку
- создание вложенной папки

---

### 2.5 Действия с файлами

#### Просмотр файла

**API запрос:** `GET /api/files/{id}/`

При клике на файл открывается страница с детальной информацией:
- имя файла
- размер
- тип (MIME)
- дата загрузки
- предпросмотр (для изображений)

#### Скачивание файла

**API запрос:** `GET /api/files/{id}/download/`

Кнопка "Скачать" на карточке файла инициирует загрузку файла на устройство пользователя.

#### Удаление файла

**API запрос:** `DELETE /api/files/{id}/`

Кнопка "Удалить" помечает файл как удалённый (мягкое удаление). Файл исчезает из списка, но сохраняется в базе.

#### Расшаривание файла

**API запрос:** `POST /api/files/{id}/share/`

Кнопка "Поделиться" создаёт публичную ссылку на файл:
- указывается время жизни ссылки (в минутах)
- указывается максимальное количество скачиваний
- генерируется уникальный токен
- ссылка копируется в буфер обмена

#### Перемещение файла

**API запрос:** `POST /api/files/{id}/move/`

Возможность переместить файл в другую папку:
- открывается список папок
- выбор целевой папки
- подтверждение перемещения

---

### 2.6 Действия с папками

#### Создание папки

**API запрос:** `POST /api/folders/`

Модальное окно с полем для ввода имени новой папки.

#### Переименование папки

**API запрос:** `PATCH /api/folders/{id}/`

Изменение имени существующей папки.

#### Удаление папки

**API запрос:** `DELETE /api/folders/{id}/`

Удаление папки вместе со всем содержимым (файлы помечаются как удалённые).

---

### 2.7 Страница профиля (Profile)

**URL:** `/profile`

**API запросы:**
- `GET /api/auth/users/me/` — получение данных
- `PATCH /api/auth/users/me/` — обновление данных
- `POST /api/auth/users/set_password/` — смена пароля

Назначение: управление данными пользователя.

Функции:
- просмотр имени пользователя и email
- изменение email
- изменение пароля (старый пароль, новый пароль, подтверждение)
- выход из аккаунта (очистка токена)

---

### 2.8 Публичный доступ (Public Link)

**URL:** `/p/:token`

**API запрос:** `GET /p/{token}/`

Назначение: доступ к файлу по публичной ссылке без авторизации.

Функции:
- проверка срока действия ссылки
- проверка лимита скачиваний
- автоматическое скачивание файла
- отображение ошибки, если ссылка истекла




## 3. API запросы

### 3.1 Авторизация (Auth)

| Метод | Эндпоинт | Описание | Тело запроса | Ответ |
|-------|----------|----------|--------------|--------|
| POST | `/api/auth/users/` | Регистрация нового пользователя | `{ "username": "user", "password": "pass" }` | Данные созданного пользователя |
| POST | `/api/auth/jwt/create/` | Получение JWT токена (вход) | `{ "username": "user", "password": "pass" }` | `{ "access": "token", "refresh": "token" }` |
| POST | `/api/auth/jwt/refresh/` | Обновление access токена | `{ "refresh": "token" }` | `{ "access": "new_token" }` |
| GET | `/api/auth/users/me/` | Информация о текущем пользователе | - | Данные пользователя |
| PATCH | `/api/auth/users/me/` | Обновление данных пользователя | `{ "email": "new@mail.com" }` | Обновлённые данные |
| POST | `/api/auth/users/set_password/` | Смена пароля | `{ "new_password": "newpass", "current_password": "oldpass" }` | `204 No Content` |

---

### 3.2 Файлы (Files)

| Метод | Эндпоинт | Описание | Параметры | Тело запроса | Ответ |
|-------|----------|----------|-----------|--------------|--------|
| GET | `/api/files/` | Список всех файлов пользователя | - | - | Массив файлов |
| POST | `/api/files/` | Загрузить один файл | - | `multipart/form-data` с полем `file` | Данные загруженного файла |
| GET | `/api/files/{id}/` | Детальная информация о файле | ID файла | - | Данные файла |
| DELETE | `/api/files/{id}/` | Удалить файл (мягкое удаление) | ID файла | - | `204 No Content` |
| GET | `/api/files/{id}/download/` | Скачать файл | ID файла | - | Файл для скачивания |
| GET | `/api/files/{id}/preview/` | Получить превью файла | ID файла | - | Изображение-превью |
| POST | `/api/files/{id}/move/` | Переместить файл в папку | ID файла | `{ "folder_id": 5 }` | `{ "message": "Файл перемещен" }` |
| POST | `/api/files/{id}/share/` | Создать публичную ссылку | ID файла | `{ "ttl_minutes": 60, "max_downloads": 5 }` | `{ "url": "/p/token/" }` |
| POST | `/api/files/bulk_upload/` | Загрузить несколько файлов | - | `multipart/form-data` с полем `files` | Массив загруженных файлов |

---

### 3.3 Папки (Folders)

| Метод | Эндпоинт | Описание | Параметры | Тело запроса | Ответ |
|-------|----------|----------|-----------|--------------|--------|
| GET | `/api/folders/` | Список всех папок пользователя | - | - | Массив папок |
| POST | `/api/folders/` | Создать новую папку | - | `{ "name": "Новая папка", "parent": null }` | Данные созданной папки |
| GET | `/api/folders/{id}/` | Информация о папке | ID папки | - | Данные папки |
| PUT/PATCH | `/api/folders/{id}/` | Обновить папку (переименовать) | ID папки | `{ "name": "Новое имя" }` | Обновлённые данные |
| DELETE | `/api/folders/{id}/` | Удалить папку | ID папки | - | `204 No Content` |
| GET | `/api/folders/{id}/content/` | Содержимое папки | ID папки | - | `{ "folders": [], "files": [] }` |

## 4. Компоненты Vue

### 4.1 App.vue (корневой компонент)

Корневой компонент приложения, содержит основную структуру и роутер.

```vue
<template>
  <div id="app">
    <NavBar v-if="!isAuthPage" />
    <router-view />
  </div>
</template>
```

**Назначение:** 
- Отображает навигационную панель на всех страницах, кроме страниц входа/регистрации
- Подключает роутер для отображения текущей страницы

---

### 4.2 NavBar (навигационная панель)

Компонент навигации, отображается на всех страницах после авторизации.

**Отображает:**
- Логотип/название приложения
- Ссылку на главную страницу (файлы)
- Ссылку на профиль пользователя
- Имя текущего пользователя
- Кнопку "Выйти"

**Функции:**
- Навигация между разделами
- Выход из аккаунта (очистка токена, редирект на логин)

---

### 4.3 FileCard (карточка файла)

Компонент для отображения файла в списке.

**Пропсы (входные данные):**
- `file` — объект с данными файла (id, name, size, mime_type, uploaded_at)

**Отображает:**
- Иконка в зависимости от типа файла (изображение, видео, pdf, документ)
- Имя файла
- Размер (в байтах/KB/MB)
- Дата загрузки
- Кнопки действий

**Кнопки действий:**
- `Скачать` — вызывает API `/api/files/{id}/download/`
- `Поделиться` — открывает модальное окно для создания ссылки
- `Удалить` — подтверждение удаления, затем вызов API `DELETE /api/files/{id}/`
- `Переместить` — открывает список папок для выбора

**События (emits):**
- `@delete` — событие удаления файла
- `@share` — событие расшаривания
- `@move` — событие перемещения

```vue
<template>
  <div class="file-card">
    <div class="file-icon">📄</div>
    <div class="file-info">
      <div class="file-name">{{ file.name }}</div>
      <div class="file-meta">{{ formatSize(file.size) }} • {{ formatDate(file.uploaded_at) }}</div>
    </div>
    <div class="file-actions">
      <button @click="downloadFile">Скачать</button>
      <button @click="shareFile">Поделиться</button>
      <button @click="deleteFile">Удалить</button>
    </div>
  </div>
</template>
```

# Лабораторная работа 1, Задание 4: Многопользовательский чат

## Описание
Многопользовательский TCP чат с использованием библиотек `socket` и `threading`.
Поддерживает одновременное подключение множества пользователей с возможностью обмена сообщениями в реальном времени.

**Реализация:** TCP протокол для получения **100% баллов**

## Файлы
- `server.py` - серверная часть чата
- `client.py` - клиентская часть чата
- `README.md` - документация

## Как запустить

### 1. Запустите сервер:
```bash
python3 server.py
```

### 2. Запустите клиентов (в отдельных терминалах):
```bash
# Терминал 1
python3 client.py

# Терминал 2  
python3 client.py

# Терминал 3
python3 client.py

# И так далее...
```

### 3. Использование:
1. Введите ваш никнейм при подключении
2. Начните отправлять сообщения
3. Для выхода введите `exit` или нажмите `Ctrl+C`

## Технические детали

### Архитектура сервера
- **Протокол:** TCP (SOCK_STREAM)
- **Порт:** 12347
- **Многопоточность:** Каждый клиент обрабатывается в отдельном потоке
- **Хранение клиентов:** Список активных подключений + словарь никнеймов
- **Рассылка сообщений:** Broadcast всем подключенным клиентам

### Архитектура клиента
- **Основной поток:** Отправка сообщений (input/send)
- **Фоновый поток:** Получение сообщений (recv в threading.Thread)
- **Синхронизация:** Глобальная переменная `running` для корректного завершения

### Особенности реализации

#### Сервер (`server.py`):
```python
# Хранение клиентов
clients = []              # Список сокетов
nicknames = {}            # Словарь сокет -> никнейм

# Обработка каждого клиента в отдельном потоке
client_thread = threading.Thread(target=handle_client, args=(client_socket, address))
client_thread.daemon = True
client_thread.start()
```

#### Клиент (`client.py`):
```python
# Получение сообщений в отдельном потоке
receive_thread = threading.Thread(target=receive_messages, args=(client_socket,))
receive_thread.daemon = True
receive_thread.start()

# Отправка в основном потоке
while running:
    message = input()
    client_socket.send(message.encode('utf-8'))
```

### ✅ Graceful disconnect
- Корректная обработка отключений
- Очистка ресурсов и уведомление других пользователей

## Тестирование многопользовательского режима

### Ожидаемое поведение:
1. Каждый новый пользователь видит уведомление о присоединении
2. Сообщения от любого пользователя видят все остальные
3. При отключении одного пользователя остальные получают уведомление

# Задание 5: Журнал оценок (веб-приложение)

## 📝 Описание

Простой веб-сервер для обработки GET и POST HTTP-запросов с помощью библиотеки socket в Python. Сервер реализует журнал оценок по дисциплинам с возможностью добавления новых оценок и просмотра всех существующих.

## 🎯 Функциональность

### Основные возможности

✅ **GET запросы**: Отображение HTML-страницы с формой для добавления оценок и таблицей всех оценок  
✅ **POST запросы**: Добавление новых оценок по дисциплинам  
✅ **Структурированное хранение**: Оценки группируются по дисциплинам (несколько оценок на одну дисциплину)  
✅ **Валидация данных**: Проверка корректности введенных оценок (1-5)  
✅ **Статистика**: Автоматический подсчет количества оценок и среднего балла  
✅ **Веб-интерфейс**: Красивая HTML-форма с CSS стилями  

## 🏗️ Технические требования

- **Протокол**: HTTP/1.1 over TCP
- **Порт**: 8080
- **Методы**: GET, POST
- **Content-Type**: application/x-www-form-urlencoded для форм
- **Хранение**: В памяти с использованием `defaultdict(list)`
- **Валидация**: Оценки в диапазоне 1-5

## 🔍 HTTP протокол в деталях

### GET запрос (просмотр журнала)

```http
GET / HTTP/1.1
Host: localhost:8080
User-Agent: Mozilla/5.0 (...)
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8
Accept-Language: ru,en-US;q=0.7,en;q=0.3
Connection: keep-alive
```

### POST запрос (добавление оценки)

```http
POST /add_grade HTTP/1.1
Host: localhost:8080
Content-Type: application/x-www-form-urlencoded
Content-Length: 31

discipline=Математика&grade=5
```

### HTTP ответ (успешное добавление)

```http
HTTP/1.1 302 Found
Location: /
Connection: close

```

## ❓ Частые вопросы

??? question "Сохраняются ли данные после перезапуска сервера?"
    В базовой версии - нет, данные хранятся в памяти. Для постоянного хранения можно добавить сохранение в JSON файл или базу данных.

??? question "Можно ли добавить несколько оценок за раз?"
    В текущей реализации - по одной. Но можно расширить форму для множественного ввода или добавить импорт из CSV.

??? question "Поддерживается ли одновременная работа нескольких пользователей?"
    Да, но без синхронизации. Для production использования нужны блокировки или база данных.

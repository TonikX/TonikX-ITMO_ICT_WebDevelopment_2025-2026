# Отчет по лабораторной работе №2 Веб-приложение "Табло победителей автогонок"

## 1. Описание задания

### Вариант 6: Табло победителей автогонок

Необходимо разработать веб-приложение для управления автогонками, которое отображает информацию об участниках гонок.

**Основные требования:**

1. **Таблица участников** должна содержать:
   - ФИО участника
   - Название команды
   - Описание автомобиля
   - Описание участника
   - Опыт и класс участника

2. **Функциональность:**
   - Регистрация новых пользователей
   - Просмотр автогонок и регистрация гонщиков
   - Система отзывов и комментариев с рейтингом (1-10)
   - Администратор может управлять результатами через Django-admin
   - Формирование таблицы всех заездов и результатов конкретной гонки

## 3. Модель данных

### 3.1. Диаграмма связей

```
User (Django built-in)
  ↓ (1:1)
Participant ──→ Team (M:1)
  ↓           └→ Car (M:1)
  ↓
RaceParticipant ──→ Race (M:1)
  
Comment ──→ Race (M:1)
        └→ User (M:1)
```

### 3.2. Описание моделей

#### **Team (Команда)**
Модель представляет гоночную команду.

**Поля:**
- `name` — название команды (уникальное)
- `country` — страна
- `founded_year` — год основания
- `description` — описание команды
- `created_at` — дата создания записи

#### **Car (Автомобиль)**
Модель описывает гоночный автомобиль.

**Поля:**
- `model_name` — название модели
- `manufacturer` — производитель
- `year` — год выпуска
- `engine_type` — тип двигателя
- `horsepower` — мощность (л.с.)
- `description` — описание
- `created_at` — дата создания записи

#### **Participant (Участник/Гонщик)**
Профиль участника гонок, связанный с пользователем.

**Поля:**
- `user` — связь с пользователем (1:1)
- `full_name` — полное имя
- `date_of_birth` — дата рождения
- `country` — страна
- `team` — команда (FK)
- `car` — автомобиль (FK)
- `experience_level` — уровень опыта (beginner/intermediate/advanced/professional)
- `bio` — биография
- `created_at` — дата создания

#### **Race (Гонка)**
Модель гоночного события.

**Поля:**
- `name` — название гонки
- `location` — локация
- `track_length` — длина трассы (км)
- `total_laps` — количество кругов
- `date` — дата и время гонки
- `status` — статус (upcoming/in_progress/completed/cancelled)
- `description` — описание
- `created_at`, `updated_at` — временные метки

#### **RaceParticipant (Участие в гонке)**
Связывает участников и гонки, хранит результаты.

**Поля:**
- `race` — гонка (FK)
- `participant` — участник (FK)
- `position` — финишная позиция
- `finish_time` — время финиша (Duration)
- `best_lap_time` — лучшее время круга (Duration)
- `points` — очки
- `dnf` — не финишировал (boolean)
- `dnf_reason` — причина DNF
- `registered_at` — дата регистрации

**Ограничения:**
- Уникальность пары (race, participant) — один участник может быть зарегистрирован в гонке только один раз

#### **Comment (Комментарий/Отзыв)**
Отзывы пользователей о гонках.

**Поля:**
- `race` — гонка (FK)
- `author` — автор (FK к User)
- `comment_type` — тип комментария (cooperation/track/other)
- `text` — текст комментария
- `rating` — рейтинг (1-10)
- `created_at`, `updated_at` — временные метки

- Создание профиля участника после регистрации

**Технические детали:**

- Хеширование паролей встроенными средствами Django
- Проверка уникальности username
- Валидация надежности пароля

### 4.2. Управление профилем

Пользователи могут:
- Создать профиль участника с указанием команды и автомобиля
- Редактировать информацию профиля
- Просматривать историю своих гонок
- Видеть статистику (позиции, очки)

### 4.3. Просмотр и фильтрация гонок

**Главная страница (index):**
- Список всех гонок в виде карточек
- **Фильтрация по:**
  - Статусу гонки (предстоящие/идущие/завершенные/отмененные)
  - Локации
  - Поисковому запросу (название, локация, описание)
- **Пагинация:** 9 гонок на страницу

**Детальная страница гонки:**
- Информация о гонке (дата, трасса, количество кругов)
- Список участников

- Комментарии и отзывы

Авторизованные пользователи могут:
- Зарегистрироваться на предстоящие гонки
- Видеть статус своей регистрации
- Просматривать список других участников

**Ограничения:**
- Регистрация возможна только на предстоящие гонки
- Один участник не может зарегистрироваться дважды на одну гонку

**Функциональность:**

- Выбор типа комментария (сотрудничество, состояние трассы, другое)

**Требования:**
- Необходима авторизация для добавления комментариев

**Визуализация:**

- Пагинация: 20 участников на страницу

### 4.7. Просмотр команд и автомобилей

## 12. Особенности реализации

### 12.1. Разделение прав пользователей

**На уровне шаблонов:**
```django

```

**На уровне views:**
```python
@login_required
@user_passes_test(is_admin)
def manage_race_results(request, race_id):
    # Только для администраторов
    ...
```

### 12.2. Обработка времени в результатах

Используется `timedelta` для хранения времени:
```python
finish_time = timedelta(hours=1, minutes=32, seconds=15)
```

Преобразование из строки:
```python
def parse_time(time_str):
    parts = time_str.split(':')
    hours, minutes, seconds = int(parts[0]), int(parts[1]), int(parts[2])
    return timedelta(hours=hours, minutes=minutes, seconds=seconds)
```

### 12.3. Вычисление статистики

Агрегация очков участника:
```python
total_points = RaceParticipant.objects.filter(
    participant=participant
).aggregate(total=models.Sum('points'))['total'] or 0
```

### 12.4. Оптимизация запросов

Использование `select_related` для уменьшения количества запросов:
```python
race_participants = RaceParticipant.objects.filter(race=race)\
    .select_related('participant', 'participant__team', 'participant__car')
```

### 16.2. Основные команды управления

```bash
# Создание миграций
python manage.py makemigrations

# Применение миграций
python manage.py migrate

# Создание суперпользователя
python manage.py createsuperuser

# Загрузка тестовых данных
python manage.py load_sample_data

# Запуск сервера разработки
python manage.py runserver

# Сбор статических файлов (для продакшена)
python manage.py collectstatic

# Проверка проекта на ошибки
python manage.py check
```

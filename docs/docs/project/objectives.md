\# Цели и задачи



\## 🎯 Цель работы



Овладеть практическими навыками и умениями реализации web-сервисов средствами Django 3.2.



\## 📋 Основные задачи



1\. \*\*Установка и настройка\*\* Django-проекта

2\. \*\*Создание приложения\*\* и базовых файлов конфигурации

3\. \*\*Реализация моделей\*\* для хранения данных

4\. \*\*Разработка представлений\*\* для логики обработки запросов

5\. \*\*Создание шаблонов\*\* (HTML-страниц с данными из модели)

6\. \*\*Настройка маршрутизации\*\* (urls.py)

7\. \*\*Подключение CSS\*\* (Bootstrap) для оформления

8\. \*\*Реализация пагинации\*\* и дополнительного функционала



\## 📊 Дополнительные требования



Для получения максимальной оценки необходимо было реализовать:



\- ✅ \*\*Меню навигации\*\* на Bootstrap

\- ✅ \*\*Пагинацию страниц\*\*

\- ✅ \*\*Поиск по объектам\*\*, совместимый с пагинацией

\- ✅ \*\*Разделение прав\*\* администратора и пользователя



\## 🏗 Архитектура проекта



Проект реализован по классической схеме \*\*MVT (Model-View-Template)\*\*:



\- \*\*Models\*\* - модели данных (Subject, Assignment, Submission)

\- \*\*Views\*\* - обработчики запросов и бизнес-логика

\- \*\*Templates\*\* - представление данных пользователю

# Инструкция по запуску

## Требования к системе

Для запуска проекта нужно установить:

- Python 3.10 или новее
- pip (менеджер пакетов Python)
- Git (опционально, для клонирования репозитория)

### Проверка версии Python

Откройте командную строку и выполните:

```bash
python --version
```

Должна показаться версия 3.10 или выше. Если у вас Python 2.x, попробуйте команду `python3`:

```bash
python3 --version
```

## Шаг 1. Получение проекта

### Вариант 1: Клонирование из Git

Если проект находится в репозитории Git:

```bash
git clone <URL-репозитория>
cd PythonProject2
```

### Вариант 2: Скачивание архива

Скачайте архив с проектом, распакуйте его и перейдите в папку проекта.

## Шаг 2. Создание виртуального окружения

Виртуальное окружение изолирует зависимости проекта от других Python проектов на вашем компьютере.

### Windows:

```bash
python -m venv .venv
.venv\Scripts\activate
```

### Linux/Mac:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

После активации в начале строки командной строки появится `(.venv)`.

## Шаг 3. Установка зависимостей

Все необходимые библиотеки перечислены в файле `requirements.txt`:

```bash
pip install -r requirements.txt
```

Это установит:

- Django (фреймворк)
- python-dotenv (для переменных окружения)
- django-crispy-forms (красивые формы)
- crispy-bootstrap5 (Bootstrap 5 для форм)

## Шаг 4. Настройка базы данных

### Создание таблиц

Django автоматически создаст все нужные таблицы:

```bash
python manage.py migrate
```

Эта команда применяет миграции - инструкции по созданию и изменению таблиц.

### Что происходит

Django создает файл `db.sqlite3` с такими таблицами:

- auth_user (пользователи)
- racing_team (команды)
- racing_driverprofile (профили водителей)
- racing_race (гонки)
- racing_heat (заезды)
- racing_heatresult (результаты)
- racing_registration (регистрации)
- racing_comment (комментарии)
- И служебные таблицы Django

## Шаг 5. Создание администратора

Для доступа в админ-панель нужно создать суперпользователя:

```bash
python manage.py createsuperuser
```

Программа попросит ввести:

- Имя пользователя (например, admin)
- Email (можно пропустить, нажав Enter)
- Пароль (вводится дважды, при вводе не отображается)

Если пароль слишком простой, Django предупредит. Можно пропустить предупреждение, введя `y`.

## Шаг 6. Создание тестовых данных (опционально)

Чтобы не заполнять базу вручную, можно сгенерировать тестовые данные:

```bash
python manage.py create_demo_data
```

Эта команда создаст:

- 6 команд гонщиков
- 10 пользователей с профилями водителей
- 10 гонок (8 опубликованных)
- Около 18 заездов
- Регистрации участников
- Результаты заездов
- Комментарии

### Тестовые пользователи

После выполнения команды можно войти под любым из созданных пользователей:

- Логин: `pilot1`, `pilot2`, ..., `pilot10`
- Пароль для всех: `demo1234`

## Шаг 7. Запуск сервера

Запустите локальный сервер разработки:

```bash
python manage.py runserver
```

Вы увидите сообщение:

```
Starting development server at http://127.0.0.1:8000/
Quit the server with CTRL-BREAK.
```

Сервер запущен! Откройте браузер и перейдите по адресу:

```
http://127.0.0.1:8000/
```

## Доступные страницы

После запуска сервера доступны:

### Для всех пользователей:

- **Главная (список гонок)**: http://127.0.0.1:8000/
- **Страница гонки**: http://127.0.0.1:8000/race/1/
- **Вход**: http://127.0.0.1:8000/login/
- **Регистрация**: http://127.0.0.1:8000/signup/

### Для авторизованных пользователей:

- **Профиль**: http://127.0.0.1:8000/profile/
- **Редактирование профиля**: http://127.0.0.1:8000/profile/update/
- **Мои регистрации**: http://127.0.0.1:8000/registrations/
- **Смена пароля**: http://127.0.0.1:8000/password_change/

### Для администраторов:

- **Админ-панель**: http://127.0.0.1:8000/admin/

## Остановка сервера

Чтобы остановить сервер, нажмите:

- **Windows**: `Ctrl + C` или `Ctrl + Break`
- **Linux/Mac**: `Ctrl + C`

## Типичные проблемы

### Ошибка "No module named 'django'"

**Причина**: Не установлены зависимости или не активировано виртуальное окружение.

**Решение**:

```bash
# Активируйте виртуальное окружение
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # Linux/Mac

# Установите зависимости
pip install -r requirements.txt
```

### Ошибка "no such table"

**Причина**: Не применены миграции.

**Решение**:

```bash
python manage.py migrate
```

### Порт 8000 уже занят

**Причина**: Другое приложение использует порт 8000.

**Решение**: Используйте другой порт:

```bash
python manage.py runserver 8001
```

### Забыли пароль администратора

**Решение**: Создайте нового суперпользователя:

```bash
python manage.py createsuperuser
```

## Дополнительные команды

### Создание новых миграций

Если вы изменили модели:

```bash
python manage.py makemigrations
python manage.py migrate
```

### Запуск интерактивной консоли

Для экспериментов с моделями:

```bash
python manage.py shell
```

### Сброс базы данных

Чтобы начать с чистой базы:

```bash
# Удалите файл базы данных
rm db.sqlite3  # Linux/Mac
del db.sqlite3  # Windows

# Примените миграции заново
python manage.py migrate

# Создайте нового администратора
python manage.py createsuperuser

# Сгенерируйте данные
python manage.py create_demo_data
```

## Что дальше?

После успешного запуска:

1. Откройте главную страницу http://127.0.0.1:8000/
2. Зарегистрируйтесь или войдите под тестовым пользователем
3. Заполните профиль водителя
4. Зарегистрируйтесь на гонку
5. Посмотрите результаты и оставьте комментарий
6. Зайдите в админ-панель http://127.0.0.1:8000/admin/

Приятного использования!

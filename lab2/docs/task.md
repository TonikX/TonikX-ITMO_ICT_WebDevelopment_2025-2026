# Задание 3. Табло отображения информации об авиаперелетах

Проект: система отображения и управления авиарейсами, бронированиями и отзывами.

## 1. Описание предметной области

Хранится информация о рейсах:
- Номер рейса
- Авиакомпания
- Время отлета
- Время прилета
- Тип рейса (прилет/отлет)
- Номер гейта
- Количество мест (для расчета доступности)

Дополнительно хранятся:
- Резервирования мест пользователями (пользователь, рейс, номер места, дата создания, активность)
- Пассажиры (привязаны к резервированию: имя, фамилия, номер билета, номер паспорта)
- Отзывы к рейсам (рейс, пользователь, дата рейса, комментарий, рейтинг 1–10, дата создания)

## 2. Технологии
- Backend: Django 5.2.7
- БД: SQLite (по умолчанию)
- Аутентификация: встроенная система Django
- Frontend: Bootstrap 5.3, Bootstrap Icons

## 3. Структура данных (модели)

1) Flight (Рейс)
- flight_number: CharField(unique)
- airline: CharField
- departure_time: DateTimeField
- arrival_time: DateTimeField
- flight_type: CharField(choices=[arrival/departure])
- gate_number: CharField
- total_seats: IntegerField (по умолчанию 100)
- available_seats(): вычисление доступных мест как total_seats − активные резервирования

2) Reservation (Резервирование)
- user: ForeignKey(User)
- flight: ForeignKey(Flight)
- seat_number: CharField(blank=True) — место необязательное
- created_at: DateTimeField(auto_now_add)
- is_active: BooleanField
- unique_together: (flight, seat_number) — предотвращает двойное занятие одного места на рейсе

3) Passenger (Пассажир)
- reservation: OneToOneField(Reservation)
- first_name, last_name: CharField
- ticket_number: CharField(unique)
- passport_number: CharField

4) Review (Отзыв)
- flight: ForeignKey(Flight)
- user: ForeignKey(User)
- flight_date: DateField
- comment: TextField
- rating: IntegerField (1–10, валидаторы)
- created_at: DateTimeField(auto_now_add)

## 4. Требуемый функционал и реализация

### 4.1 Регистрация новых пользователей
- Реализовано через форму UserRegistrationForm (на основе UserCreationForm)
- URL: /register/
- После регистрации — автоматический вход и редирект на список рейсов
- Встроенные URL аутентификации: /accounts/login/, /accounts/logout/

### 4.2 Просмотр и резервирование мест на рейсах
- Список рейсов: главная страница / (flights/flight_list.html) — таблица всех рейсов, доступные места, средний рейтинг
- Детальная страница рейса: /flight/<id>/ — полная информация, отзывы, кнопки «Забронировать место», «Оставить отзыв», «Список пассажиров»
- Создание резервирования: /flight/<id>/reserve/
  - Перед созданием проверяется: нет ли уже активного резервирования у пользователя; есть ли свободные места
  - seat_number — необязателен. Можно забронировать без выбора места — в интерфейсе показывается «место не выбрано»
  - Интерактивная схема мест: 20 рядов, по 2 слева и 2 справа (A,B — проход — C,D)
    - Свободные места — синие, занятые — красные, выбранные — зеленые
    - Клик по месту автоматически подставляет номер (например, 12A)
- Редактирование резервирования: /reservation/<id>/edit/
  - Также показывает интерактивную схему мест
  - Текущее место подсвечено оранжевым; новый выбор — зелёным; занятые — красным
- Удаление (отмена) резервирования: /reservation/<id>/delete/ (фактически, деактивация is_active=false)
- Список собственных бронирований: /reservations/ — карточки с рейсами, кнопки «Изменить», «Отменить», «Зарегистрироваться на рейс»

Валидации и ограничения:
- Нельзя создать два активных бронирования на один рейс для одного пользователя
- Нельзя выбрать место, уже занятое другим активным бронированием на этом рейсе
- Пользователь может редактировать и удалять только свои резервирования

### 4.3 Регистрация пассажира администратором (Django Admin)
- В админке настроены модели Flight, Reservation (с inline Passenger), Passenger, Review
- Администратор может открыть конкретное резервирование и в inline-блоке Passenger заполнить данные пассажира, включая номер билета
- Также доступен отдельный пользовательский путь регистрации пассажира: /reservation/<id>/register-passenger/
  - Упрощает самостоятельную регистрацию на рейс: форма с полями (имя, фамилия, номер билета, паспорт)

### 4.4 Таблица всех пассажиров рейса (клиентская часть)
- URL: /flight/<id>/passengers/
- Таблица с колонками: №, Имя, Фамилия, Номер билета, Номер паспорта, Место, Пользователь
- Отображаются данные всех активных пассажиров по данному рейсу

### 4.5 Отзывы к рейсам
- URL добавления: /flight/<id>/review/
- Поля: дата рейса, комментарий, рейтинг (1–10)
- Автозаполнение даты рейса — подставляется дата отлёта рейса (input type=date, формат YYYY-MM-DD)
- На детальной странице рейса отображается список отзывов и средний рейтинг по рейсу

## 5. Пользовательские интерфейсы
- Навигация в шапке: «Все рейсы», «Мои бронирования», вход/выход, регистрация
- Сообщения об успехе/ошибках (Django messages) в верхней части контента
- Адаптивные таблицы и карточки (Bootstrap)
- Интерактивная схема мест (JavaScript): генерация разметки рядов/мест, подсветка состояний, установка значения в поле формы

Цветовая индикация:
- Свободные места: синие
- Занятые: красные
- Выбранные: зелёные
- Текущее место при редактировании: оранжевое

Отображение места в бронированиях:
- Если seat_number пустой — выводится «место не выбрано»

## 6. URL-маршруты
- / — список рейсов
- /register/ — регистрация пользователя
- /accounts/login/, /accounts/logout/ — вход/выход
- /flight/<id>/ — детальная страница рейса
- /flight/<id>/reserve/ — бронирование
- /flight/<id>/passengers/ — пассажиры рейса
- /flight/<id>/review/ — добавить отзыв
- /reservations/ — мои бронирования
- /reservation/<id>/edit/ — редактирование бронирования
- /reservation/<id>/delete/ — отмена бронирования
- /reservation/<id>/register-passenger/ — регистрация пассажира (клиентский путь)
- /admin/ — панель администратора

## 7. Безопасность и права доступа
- Действия по созданию/редактированию/удалению резервирований — только для аутентифицированных пользователей
- Просмотр «Мои бронирования» — только свои записи
- Регистрация пассажира через клиентскую форму доступна только автору соответствующего активного резервирования
- Админ-панель — только для суперпользователей/персонала

## 8. Инициализация и запуск

Установка зависимостей:
```
pip install django
```

Миграции БД:
```
python manage.py makemigrations
python manage.py migrate
```

Создание суперпользователя:
```
python manage.py createsuperuser
```

Загрузка тестовых данных (5 рейсов):
```
python manage.py populate_data
```

Запуск сервера разработки:
```
python manage.py runserver
```

Адреса:
- Приложение: http://127.0.0.1:8000/
- Админка: http://127.0.0.1:8000/admin/
